# 能力系统索引

状态：采用
基准：`../../02_总纲/以太纪元世界观总纲.md`

本文件是“以太纪元”能力系统专题的导航入口。能力系统下各文件保持同级关系，不将架构演变、标准化体系、灵脉谱系或职业模板降为个体能力运行机制的子章节。

## 文件分工

| 文件 | 状态 | 主要职责 |
| --- | --- | --- |
| `个体能力运行与成长机制.md` | 采用 | 说明灵核、灵脉、接口灵脉和灵械如何在个体身上运行、获取、建立、共适应、训练、管控、修复、抹除、变更、失效和成长。 |
| `灵脉架构演变.md` | 采用 | 说明能力模块如何从早期体内堆叠演变为现代接口主导、体内核心与外置灵械分工协同的架构。 |
| `灵脉与灵械标准化体系.md` | 采用 | 说明能力结构如何被工程化、许可、认证、维护、监管、管控、商品化、军用化和定制化。 |
| `灵脉谱系设定.md` | 采用 | 说明灵脉结构如何从原型发展为主干、分支、旁支、断代残支或遗产接枝。 |
| `职业与灵脉模板库.md` | 草稿 | 说明能力系统如何落到职业配置、训练路径、灵械套件和角色应用。 |

## 边界规则

- 运行机制回答“能力如何在个体身上工作、获得、管控、修复、变更或抹除”。
- 架构演变回答“能力模块如何在体内核心、接口灵脉、外置灵械和团队系统之间分工”。
- 标准化体系回答“能力如何被测试、许可、认证、维护、交易、管控和监管”。
- 谱系体系回答“能力结构从哪里来，继承了什么，又如何分化”。
- 职业模板回答“某类人如何把这些能力组合成可训练、可维护的职业架构”。

新增能力设定时，先判断它主要改变哪一类问题；如果同时影响多类问题，应在主文件中写正文，并在其他相关文件中补充引用或边界说明。
