# 灵脉谱系设定

状态：采用  
基准：`../../02_总纲/以太纪元世界观总纲.md`、`能力系统索引.md`、`个体能力运行与成长机制.md`、`灵脉与灵械标准化体系.md`

本文件是“以太纪元”灵脉谱系的权威扩写入口。个体能力运行与成长机制负责说明灵脉如何在个体身上运行；标准化体系负责说明灵脉如何被工程化、认证和定制；本文件负责说明不同灵脉之间如何继承、分叉、融合、断代和重新接入现有体系。

## 一、谱系定位

灵脉谱系是对灵脉发展脉络的记录，不是新的固定能力分类。它用于追踪一条灵脉从原型结构发展为多个分支的过程，以及这些分支之间共享了哪些核心逻辑、改变了哪些关键结构、适配了哪些承载对象，并留下了哪些兼容风险。

谱系只说明发展关系，不直接决定灵脉强弱、稀有度、标准化状态或定制化状态。同一条谱系可能同时存在标准教学分支、军用改型、家族封存分支和黑市旁支；同一功能也可能由不同谱系独立实现。

写作时应区分三组概念：

- 功能分类：说明灵脉做什么，例如感知、认知、操控、生理或特殊功能。
- 标准状态：说明灵脉是否公开、可认证、可维护、可教学、可交易。
- 谱系关系：说明灵脉从哪里来，继承了什么，分化到哪里，以及和哪些分支兼容或冲突。

因此，红外视觉既可以是感知类灵脉，也可以有民用标准分支、侦察职业分支和军用识别分支；这些标签描述的是不同维度，不能互相替代。

## 二、核心术语

| 术语 | 定义 | 写作用途 |
| --- | --- | --- |
| 原型灵脉 | 一条谱系中最早被稳定记录、复现或追溯的结构源头。 | 说明后续分支继承的基础逻辑。 |
| 谱系主干 | 由原型灵脉延续出的主要发展路线，通常保留核心结构和训练方式。 | 说明某类灵脉的主流教学、维护或研究路径。 |
| 分支灵脉 | 从主干中分化出的稳定改型，已形成明确适配对象和维护方式。 | 用于职业、文明、家族或任务差异。 |
| 旁支改型 | 偏离主干较多，但仍能追溯到同一原型逻辑的改造版本。 | 用于黑市改装、个人定制或地方工坊版本。 |
| 断代谱系 | 原型资料、维护方法或训练路径中断，只能通过残留结构复原的谱系。 | 用于遗产、失传家族技术和灾后残存灵脉。 |
| 遗产接枝 | 将古文明蓝图、深层以太回响或未知模块接入现有灵脉谱系的行为。 | 用于高风险能力跃迁和文明伦理冲突。 |
| 谱系污染 | 分支引入异常结构后，反向影响同源灵脉或接口系统的风险。 | 用于解释兼容失败、回响感染和肃正协议警戒。 |

“原创灵脉”只描述设计来源，不等于新谱系已经成立。只有当它的原型逻辑被记录、复现，并出现可追踪的分支、维护方法或教学资料时，才可以被整理为一条谱系。

## 三、发展机制

灵脉谱系通常通过以下机制形成：

- 自然形成：某些种族、家族或特殊环境中的灵核长期出现相近结构，后续被整理为可描述的原型灵脉。
- 训练诱导：职业学校、军校、宗派式训练团体或工会通过稳定训练路径，让相似灵脉结构在不同个体中重复出现。
- 工程铭刻：灵脉工程师通过结构模型、刻录工具和维护记录，把能力原型转化为可复现的铭刻方案。
- 职业标准化：某条灵脉在特定职业中被长期验证，形成主干教材、接口要求、维护周期和失效处理方式。
- 家族长期适配：家族或血脉群体围绕自身灵核倾向不断微调灵脉，使其对内部成员稳定，对外部个体兼容性较差。
- 任务压力分化：深层以太潜航、污染区救援、舰队登陆、遗产隔离等任务会催生短期高适配分支。
- 古文明蓝图复原：遗产资料被现代体系转译后，可能成为断代谱系的复原起点，也可能通过遗产接枝改变现有主干。

谱系发展不是单向进步。一条分支可能因为维护成本过高被封存，也可能因战时需求重新启用；某个旁支改型可能在长期验证后转入标准体系；遗产接枝可能带来性能跃迁，也可能造成污染、人格回响或肃正协议警戒。

## 四、谱系记录字段

记录一条灵脉谱系时，至少说明以下内容：

| 字段 | 说明 |
| --- | --- |
| 谱系名称 | 使用世界内名称，不用临时比喻。 |
| 原型逻辑 | 原型灵脉解决什么能力问题，核心输入、处理和输出是什么。 |
| 关键结构 | 保持谱系身份的核心灵纹、灵环、灵节或灵段。 |
| 发展来源 | 自然形成、训练诱导、工程铭刻、职业标准化、家族适配、任务分化或遗产复原。 |
| 主要分支 | 稳定存在的分支灵脉、旁支改型或断代残支。 |
| 适配对象 | 适合哪些灵核参数、种族倾向、职业任务、灵械接口或文明制度。 |
| 公开程度 | 公开教材、军方封存、家族秘传、商盟授权、环带限制或遗产隔离。 |
| 维护体系 | 谁能维护，维护周期如何，故障如何定位、降级或断连。 |
| 兼容风险 | 同源兼容、频率冲突、接口污染、认知过载或谱系污染。 |
| 失效模式 | 失效时表现为降级、延迟、误读、烧蚀、污染、人格回响或外部劫持。 |

如果谱系尚未稳定，应标明“未成谱系”“断代待复原”或“实验分支”，避免把短期定制方案写成成熟传承。

## 五、与标准化和定制化的关系

谱系关系可以帮助判断定制方案的结构来源，但不能替代标准化审核。

- 标准灵脉可以来自成熟谱系，也可以来自曾经的原创设计。
- 定制灵脉可以基于成熟谱系改造，也可以来自尚未成谱系的原创方案。
- 参考灵脉通常属于某条谱系的原型、主干或分支。
- 遗产型参考灵脉可能属于断代谱系，也可能只是无法归谱的未知结构。

同源谱系通常更容易同步和维护，但不代表绝对安全。两个同源分支如果长期向不同任务压力演化，可能在运行频率、反馈强度、接口协议或污染隔离方式上产生冲突。相反，不同谱系的灵脉如果通过接口标准和调频结构处理，也可以稳定协同。

## 六、示例谱系

### 红外视觉谱系

原型逻辑：捕捉热源差异，将不可直接识别的热信号转译为可处理的视觉或意识层信息。

主要发展脉络：

- 基础热源感知：低功率、低分辨率，适合早期训练和民用安全场景。
- 民用夜视分支：加入安全阈值和过曝保护，可由外置目镜承载，维护成本低。
- 侦察多谱段感知分支：与声波定位、残热追踪和环境滤波协同，注意力窗口占用更高。
- 军用目标识别前端：将热源轮廓接入目标识别、弹道预测和战术网络，依赖高等级接口灵脉与权限审计。

主要风险：高灵敏版本容易造成感知残留；军用分支若与战术推演灵脉延迟不一致，可能出现目标误锁或反馈过载。

### 接口灵脉谱系

原型逻辑：在灵核与外置灵脉、灵械或网络系统之间建立识别、同步、转译、保护和断连路径。

主要发展脉络：

- 通用接入分支：低风险、低权限，适合基础灵械和职业训练。
- 舰队同步分支：强化多节点同步、指令校验和战损断连，适合联邦舰队体系。
- 灵械工坊维护分支：重视诊断、调频、替换件识别和故障复测，适合灵脉工程师与模块设计师。
- 遗产隔离接口分支：加入沙箱接入、单向转译、污染熔断和人工复核，用于未知蓝图和深层以太装置。

主要风险：接口谱系越高级，越容易成为污染、劫持和权限争夺的入口。遗产隔离分支不应被写成万能钥匙，它只能降低风险，不能消除未知结构的代价。

### 先驱者遗产谱系

原型逻辑：来自先驱者蓝图或深层以太回响，现代文明只能通过残留结构、异常反馈和沙箱复原推测其能力链条。

主要发展脉络：

- 未知蓝图：原型不完整，识别困难，不能直接铭刻。
- 沙箱复原分支：在隔离环境中复现局部灵段，只允许单向转译和人工复核。
- 联邦封存分支：作为军用与遗产研究资产保存，重点是风险审计和舰队级隔离。
- 裂谷人体实验分支：尝试直接适配改造个体，性能上限高，污染和人格回响风险极高。
- 商盟商品化尝试：试图拆分为可授权模块，但容易引发契约争议、灰色改装和肃正协议警戒。

主要风险：遗产谱系的断代程度越高，现代分支越可能误读原型逻辑。能力跃迁必须伴随识别困难、接口不兼容、污染风险或文明监管冲突。

### 赤谱感知谱系

原型来源：早期热源感知灵脉。它的原型不是“感知类”这个功能分类，而是一组可复现的赤谱感应环、背景滤波环和视觉转译段，用于把热差异转成可读的轮廓信息。

主干演化：

- 原型记录：由低分辨率热源感知训练发展而来，只能辨认近距离热源和大块温差。
- 教学主干：加入过曝保护、基础降噪和低功率调控，成为军校、工会和民用安全机构可教学的基础赤谱灵脉。
- 民用外置分支：把主要感应段迁移到外置目镜，牺牲反应速度，换取低侵入、易维护和可替换镜组。
- 侦察多谱段分支：接入声波定位、残热追踪和环境滤波，强化目标搜寻，但显著增加注意力窗口占用。
- 军用识别前端分支：将赤谱轮廓输出给目标识别、弹道预测和战术网络，依赖高权限接口和延迟校准。
- 黑市过敏旁支：非法提高感应阈值，能捕捉微弱热残留，但容易造成感知残留、误锁和神经疲劳。

谱系身份：各分支都保留赤谱感应环和热差转译段，因此能追溯到同一原型；如果只使用战术网络进行目标识别，而不继承赤谱感应结构，就不属于本谱系。

维护与失效：主干维护重点是滤波和过曝保护；军用分支重点是接口延迟；黑市旁支常见问题是放大段烧蚀和感知残留。

### 通用接口谱系

原型来源：早期外置灵械接入灵脉。原型目标是让灵核识别外置模块、建立同步频率、限制反馈强度，并在异常时断连。

主干演化：

- 原型接入段：只支持低权限外置模块，能识别模块存在和基础反馈。
- 通用接入主干：加入身份识别、同步频率、反馈限流和手动断连，成为职业训练中的基础接口灵脉。
- 舰队同步分支：强化多节点同步、指令校验、权限分级和战损断连，适合联邦舰队体系。
- 灵械工坊维护分支：强化诊断、调频、替换件识别和故障复测，适合灵械模块设计师。
- 商盟授权分支：加入授权密钥、交易记录和售后追责结构，适合跨文明商品模块。
- 遗产隔离分支：加入沙箱接入、单向转译、污染熔断和人工复核，用于未知蓝图和深层以太装置。

谱系身份：各分支都继承识别、同步、限流、断连这组核心结构。只要某个灵脉只提供通讯或权限记录，而不承担灵核与外置模块的同步保护，就不能归入本谱系。

维护与失效：通用主干常见问题是同步漂移；舰队分支风险是指令串扰；商盟分支风险是密钥伪造；遗产隔离分支风险是接口倒灌和污染熔断失败。

### 玄庭推演谱系

原型来源：联邦早期航路演算训练灵脉。原型并不是所有“战术推演”能力，而是一套以空间定位、航迹残留、威胁权重和注意力分屏为核心的推演结构，最初用于小规模舰队的航路规避。

主干演化：

- 航路规避原型：根据星体、残骸、以太扰动和舰队位置计算安全路径。
- 军校教学主干：加入威胁权重、火力窗口和撤离顺序，形成舰队指挥训练中的标准推演灵脉。
- 远征舰队分支：强化长时间态势维持、跨舰同步和异常环境漂移修正。
- 特战小队分支：裁剪大型舰队结构，保留短时路径预测、队友位置和近距威胁提示。
- 名门封存分支：部分军事家族保留旧式权重模型，适合特定训练风格，但与新式舰队接口兼容较差。
- 先驱者接枝实验：尝试把遗产蓝图中的异常预测段接入主干，可能提升未知态势识别，也可能引发人格回响和谱系污染。

谱系身份：本谱系的核心不是“会推演”，而是继承航路规避原型中的空间权重、路径压缩和注意力分屏结构。医疗诊断、工程排障或商盟调度即使也需要推演，若没有继承这些结构，就不属于玄庭推演谱系。

维护与失效：主干需要清理旧战例权重和失败路径残留；名门分支容易产生训练风格固化；遗产接枝实验需要沙箱和人工复核，不能直接作为标准升级。

### 霜膜护层谱系

原型来源：环带生态维护者用于保护微型生态舱的低功率隔离膜。原型目标是隔开污染、温差和小型冲击，而不是正面承受战斗火力。

主干演化：

- 生态舱隔离原型：低功率、稳定、可长期运行，适合保护脆弱生物样本。
- 环带维护主干：加入污染识别、缓慢修复和生态阈值限制，成为环带保守标准的一部分。
- 医疗隔离分支：用于病患、灵核裂纹和神经反噬后的低刺激防护。
- 深层封存分支：强化污染隔离和单向排放，用于遗产残片、深层以太样本和危险灵械。
- 联邦战场改型：提高展开速度和抗冲击能力，但牺牲长期稳定和生态友好阈值。
- 黑市硬化旁支：强行提高承压上限，短时接近战术护盾，但容易导致护层破裂后污染反冲。

谱系身份：各分支都继承霜膜式缓冲和污染隔离结构。纯粹依靠高吞吐输出硬抗冲击的战斗护盾不属于本谱系，最多只能借鉴其中的隔离段。

维护与失效：环带主干重视长期稳定；战场改型重视战损降级；黑市旁支常见问题是稳定环烧蚀和污染倒灌。

## 七、写作用途

灵脉谱系适合用于以下场景：

- 角色能力卡：说明角色的核心灵脉来自哪条谱系，继承了什么，偏离了什么。
- 职业模板：说明职业标配灵脉的主干来源，以及允许哪些安全分支。
- 家族与文明差异：说明某个家族、军方、商盟工坊或裂谷实验室为何形成独特能力路线。
- 剧情冲突：通过谱系断代、遗产接枝、分支争夺、公开程度和维护权归属推动事件。
- 维护与事故调查：通过追溯分支来源判断故障来自原型缺陷、定制改写、接口污染还是非法旁支。

新增灵脉谱系时至少检查：

- 它的原型逻辑是否清楚。
- 它是成熟谱系、实验分支、断代谱系，还是尚未成谱系的原创设计。
- 它和既有功能分类、标准状态、定制逻辑是否混淆。
- 它的主要分支如何继承或偏离原型结构。
- 它适配哪些灵核参数、职业任务、文明制度或灵械接口。
- 它的公开程度、维护体系和失效模式是什么。
- 它是否带来同源兼容、频率冲突、接口污染、认知过载或谱系污染。
