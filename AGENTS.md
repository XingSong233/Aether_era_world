# 世界观创作指南

本文件用于指导“以太纪元”世界观后续创作、整理和扩写。它不是世界观正文，而是写作规则、设定维护规则和创作检查清单。

## 一、文件定位

- `01_原始资料/废弃/Aether_era_world.md`：废弃追溯用原始世界观总设定资料。
- `01_原始资料/废弃/灵脉设计与职业体系.md`：废弃追溯用原始能力系统深化资料。
- `02_总纲/以太纪元世界观总纲.md`：由两个原始 Markdown 合并整理出的当前总纲，也是当前权威设定入口。
- `03_专题设定/`：术语、角色、文明、剧情等专题扩写文件。
- `03_专题设定/能力系统/能力系统索引.md`：当前能力系统专题导航入口。
- `03_专题设定/能力系统/个体能力运行与成长机制.md`：当前个体能力运行、建立、训练、失效和成长机制的权威扩写入口。
- `03_专题设定/深层以太设定.md`：当前深层以太定位、组织化探查理由、个体进入风险和剧情用途的权威扩写入口。
- `04_版本记录/版本索引.md`：稳定版本索引。
- `04_版本记录/CHANGELOG.md`：项目变更记录。
- `AGENTS.md`：创作指南和维护规则。

后续创作应优先参考 `02_总纲/以太纪元世界观总纲.md`。涉及能力系统整体边界时，先参考 `03_专题设定/能力系统/能力系统索引.md`；涉及个体能力、灵核、灵脉、灵械和职业能力构建时，参考 `03_专题设定/能力系统/个体能力运行与成长机制.md`；涉及深层以太探查、风险、样本、回响和文明争夺时，参考 `03_专题设定/深层以太设定.md`。如果要追溯早期来源，再查看 `01_原始资料/废弃/` 中的两个原始 Markdown 文件。

## 二、目录操作规则

- 禁止批量删除文件或目录。
- 不要使用：
  - `del /s`
  - `rd /s`
  - `/rmdir /s`
  - `Remove-Item -Recurse`
  - `rm -rf`
- 需要删除文件时，只能一次删除一个明确路径的文件，例如：
  - `Remove-Item "C:\path\to\file.txt"`
- 如果需要批量删除文件，应停止操作，并请求用户手动删除。
- 不要直接删除原始设定文件。需要重组时，应新增整理文件或先征求用户确认。

## 三、创作核心原则

- 科技感优先于神秘感：灵核、灵脉、灵械应被写成可研究、可设计、可维护的工程体系，而不是不可解释的魔法。
- 限制比能力更重要：任何能力都应有容量、算力、吞吐、稳定性、注意力或接口带宽方面的代价。
- 文明差异要落到制度和技术路线：不同文明不只是审美不同，还应在风险态度、职业结构、法律、伦理和产业上不同。
- 星海应保持未知感：即使文明已经星际化，暗区、深层以太、古文明遗产和肃正协议仍应制造边界感。
- 主角强大来自架构能力：沧月心悚的优势不是“最强单一灵脉”，而是灵核兼容性、可塑性、推演能力和复杂系统整合能力。
- 世界观设定不应被主线剧情牵引。本项目的优先目标是创造一个能自行运转的世界，而不是为某条故事线服务。
- 扩写设定时，应优先从以太、灵核、灵脉、灵械、深层以太、文明制度、产业结构等底层运行规律出发，推导自然形成的社会情况、职业分工、技术路线、风险治理和文明冲突。
- 主角、剧情和事件线只能作为世界运行结果的样例或观测切面，不应成为设定扩写的主要驱动力。

## 四、统一术语口径

- 以太：能量、信息和意识共振的基础介质。
- 灵核：智慧生命与以太交互的核心器官。
- 灵脉：封装具体能力的以太信息回路，可训练、诱导或铭刻。
- 灵纹、灵环、灵节、灵段：灵脉从微观到宏观的结构层级。
- 接口灵脉：连接灵核与外置灵脉或灵械的通用灵脉。
- 灵械：外置的人工灵脉装置，用于承载高功率、高风险或大型能力模块。
- 共鸣网络：基于以太的信息通讯网络。
- 深层以太：以太层更深处的异常领域，场强、信号密度、反馈速度和结构复杂度远超稳定区以太环境。
- 肃正协议：古文明遗留的自动监管系统，会封锁或清除危险技术与灾害。

## 五、能力系统写作规则

设计角色能力时，至少检查以下问题：

- 这项能力依赖哪些灵核参数？
- 它占用多少结构空间、算力线程、吞吐通道和注意力窗口？
- 它是体内灵脉、外置灵械，还是二者协同？
- 它的副作用是什么？
- 它如何训练、维护、升级或失效？
- 它是否会改变角色性格、职业选择或社会地位？

避免写法：

- 单纯用“天赋强”解释能力。
- 给角色添加能力但不说明代价。
- 把灵械写成普通装备，而忽略接口、带宽、延迟和维护。
- 让古文明遗产成为无成本外挂。

推荐写法：

- 用灵核六维参数解释个体差异。
- 用灵脉架构解释能力组合。
- 用灵械和接口解释现代化、工业化、团队化能力。
- 用副作用和伦理争议推动剧情。

## 六、文明写作规则

写文明圈时，应至少明确：

- 核心理念。
- 政治结构。
- 技术路线。
- 对灵核改造和灵械普及的态度。
- 代表职业。
- 与其他文明的冲突点。

现有文明口径：

- 星海联邦：开拓、共存、遗产共享，重视军用标准化和舰队协同。
- 永恒环带：稳定、秩序、生态维护，限制危险技术。
- 深渊裂谷：进化、改造、深层以太探索，接受高风险实验。
- 商盟：契约、流通、模块化市场，推动灵械商品化。

## 七、角色写作规则

角色设定应同时包含：

- 种族与文明背景。
- 灵核六维参数倾向。
- 已有灵脉和灵械。
- 职业训练路线。
- 能力代价和心理影响。
- 与主要文明矛盾的关系。

主角沧月心悚的写作重点：

- 她是狐人族军校生，出身衰落军事名门。
- 她适合指挥、推演、感知和复杂系统整合。
- 她的核心优势是兼容性、可塑性和多系统协同，而不是单一破坏力。
- 她与先驱者遗产、深层以太和古代回响有关。
- 她的成长线应同时面对家族期待、能力失控、文明伦理冲突和星海未知。

## 八、剧情设计方向

推荐主线冲突：

- 联邦远征舰队探索过渡区或暗区。
- 主角接触古文明灵脉蓝图，引发灵核异常。
- 深潜议会试图夺取并改造该蓝图。
- 永恒环带认为该技术可能触发大寂灭风险。
- 商盟试图将遗产能力模块商品化。
- 肃正协议或共鸣虫群被遗产活动唤醒。
- 失落星庭的线索逐步浮现。

剧情应让“力量”和“代价”同时增长。每次能力升级都应带来新风险、新责任或新敌人。

## 九、设定维护规则

- 新增设定时，优先判断它属于宏观世界观、能力系统、角色、文明、剧情还是术语。
- 如果新设定改变核心规则，应同步更新 `02_总纲/以太纪元世界观总纲.md`。
- 如果新设定只是细节扩写，可先放入对应专题文件，之后再整理进总纲。
- 如果出现冲突，不要直接删除旧设定；先标明采用版本和废弃原因。
- 文档统一使用 UTF-8 编码。
- 读取 `.md`、`.txt` 等中文设定文件时，必须显式指定 UTF-8 编码，避免 PowerShell 或终端默认编码导致乱码、误读或错误判断。
- 在 PowerShell 中读取中文设定文件时，优先使用 `Get-Content -Encoding UTF8 "路径"`。
- 如果读取结果出现乱码、异常字符或疑似编码错误，应先用 UTF-8 重新读取并确认，不得直接判断文件内容损坏或正文存在问题。
- 写入或更新文档时继续保持 UTF-8 编码。
- 重要变更应同步更新 `04_版本记录/CHANGELOG.md`。
- 稳定节点应登记到 `04_版本记录/版本索引.md`。

## 十、创作复盘与错误修正规则

后续扩写设定时，应避免本次“灵脉与灵械标准化体系”补充中出现过的错误：

- 不要使用“像现实中的某技术”“类似现实发明”这类外部类比作为世界观正文表述。用户用现实概念沟通时，应转译成世界内语言，例如“能力假设、结构模型、验证、公开、普及、维护体系”。
- 不要把用户用于沟通的临时说法直接写进设定。写入前应判断它是世界内术语、解释性比喻，还是仅用于沟通的辅助表达。
- 不要把“原创灵脉”误写成固定分类。原创只描述设计来源，即基本不依赖现有灵脉结构而形成的新设计；它后续属于标准灵脉还是定制化灵脉，取决于技术资料公开状况、结构稳定性、适配范围、教学维护体系和普及程度。
- 不要把灵脉功能分类误写成发展谱系。功能分类只回答“这条灵脉做什么”，例如感知、推演、防护、通讯、牵引；发展谱系必须回答“它从哪个原型来，主干如何继承，分支如何偏移，旁支或断代风险是什么”。如果只能写用途、职业适配和资源压力，应放入能力体系或职业模板，不应命名为某某谱系。
- 不要把“适合某机构”与“由某机构或活动产生”混淆。研究机构、灵脉工程师、古文明复原项目或前沿军工实验会产生原创设计或尚未被发明的灵脉，不等于原创设计只适合这些对象。
- 不要把具体执行流程写成唯一规则。定制灵脉的执行环节会随实施者、机构、文明制度和任务目标变化；设定文件应优先写清结构来源、适配对象、取舍逻辑、风险和归类标准。
- 使用“范式”“流程”“体系”“逻辑”等抽象词前，应确认词义是否准确。如果词语会造成误导，优先使用更直接的标题，如“定制逻辑”“归类标准”“结构来源”。
- 修改核心口径后，应同步检索旧词和旧句，避免同一项目中同时残留新旧表述。

当用户指出创作错误、设定误读或表达口径问题时，应执行以下维护动作：

- 先修正文档正文，再检查相关入口文件、版本索引和 CHANGELOG 是否仍保留旧口径。
- 将可复用的教训总结为 AGENTS.md 中的写作规则或维护规则，避免下次重复。
- 总结错误时应写成可执行检查项，不写成泛泛的自我评价。
- 更新 AGENTS.md 时不得删除既有规则；如需重组，应保留原意并优先追加或小范围改写。
- 如果错误涉及核心设定分类、术语口径或维护流程，应同步考虑是否需要更新总纲、专题文件和版本记录。

## 十一、后续优先任务

- 优先完善文明圈对照表，重点补充政治结构、监管制度、产业组织、教育体系、医疗体系和军工体系。
- 优先建立职业与社会分工专题，包括职业与灵脉模板库、灵核医学、灵脉工程、灵械维护认证、事故调查、遗产隔离和黑市改装等社会角色。
- 继续扩写术语表，统一灵纹、灵环、灵节、灵段、接口带宽、注意力窗口、标准规格、遗产隔离和共鸣网络等概念。
- 建立技术史或制度史专题，用于记录灵脉自然生成、铭刻技术、灵械工业、接口标准、军用管制、深层以太事故、商盟模块市场和环带封存体系等世界演化过程。
- 主角设定卡和剧情年表可以保留并继续整理，但不作为当前世界观扩写的主要牵引项。
- 将 `.docx` 是否需要同步更新作为单独任务处理。
