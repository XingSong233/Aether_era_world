# 星海疆域生成器

状态：工作台辅助程序，原型

本目录用于按随机数种子和可调参数生成“主星图显示节点”的文明扩张结果。它不是正式世界观正文，也不直接替代 `../星海地理与星图节点设计蓝图.md` 或 `../../../03_专题设定/星际航行和开拓秩序/星海地理与星图节点.md`。

## 文件

- `index.html`：浏览器入口，直接打开即可运行。
- `styles.css`：工作台界面样式。
- `src/simulator.js`：节点生成、航路构图、深层以太区、以太洋流、星门、遗迹和文明扩张模拟算法。

## 当前规则

- 由一个随机数种子和当前参数共同控制初始状态；同一种子、同一组参数会复现同一张图。
- 默认生成 `450` 个主星图显示节点，可切换“悬臂密度场”或“均匀随机”分布。
- 先按邻近关系生成基础航路，并补足全图连通。
- 随机生成不规则深层以太区，区内节点开发阈值上升，穿越航路成本上升。
- 随机生成连续以太洋流路径，顺流和逆流成本可分别调节。
- 随机选择星门节点，已归属星门可向无主星门输送影响力。
- 随机选择古代文明遗迹节点；遗迹被纳入文明后，按递减公式微幅提升该文明整体输送效率。
- 四文明起点从高适宜候选节点中选出，并保留最小间距约束。
- 文明节点持续向相邻无主节点输送影响力；影响力达到开发阈值后，该节点转为对应文明疆域，并按产能爬升参数逐步成为稳定输出点。

## 使用

1. 打开 `index.html`。
2. 修改随机数种子或参数。
3. 点击“生成并运行”。
4. 拖动底部迭代条，或在“步数”输入框中输入指定步数。
5. 点击“导出 JSON”保存节点、航路、环境区、起点和每一轮历史快照。

浏览器会尝试把最近一次运行写入 `localStorage`。如果参数过大导致本地缓存空间不足，仍可通过“导出 JSON”保存完整结果。

## 输出口径

导出的 `history` 保存每次迭代的节点归属、节点年龄、四文明影响力、争议状态、开发进度和文明统计。后续若要整理为正式星图，应先筛选结果，再把稳定节点写入结构化地图数据或正式专题；不要把随机生成结果未经整理直接视为权威设定。
