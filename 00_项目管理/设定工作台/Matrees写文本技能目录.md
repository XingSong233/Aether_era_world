# Matrees 写文本技能目录

本文件保存当前项目内使用的 Matrees 写文本与富文本发布技能目录。它是 `C:/Users/KeHao/.codex/skills/matrees-world-richtext/SKILL.md` 的项目内镜像和维护入口，用于在本项目中追溯写作风格、页面改写规则和 Matrees 富文本发布注意事项。

本文件不是世界观正文专题，不直接产生世界内设定。稳定设定仍以 `../../02_总纲/以太纪元世界观总纲.md` 和 `../../03_专题设定/` 下的正式专题为准；本文件只规定“怎样改写、怎样发布、怎样选择文本风格”。

## 一、使用场景

当需要把本地 Markdown、设定笔记、世界观页面、时间线、作品片段或世界内文本整理到 Matrees 时，优先使用本技能目录。目标是把原文改写成网站原生富文本，而不是把 Markdown 标记原样贴到页面里。

适用任务包括：

- 世界概念、世界简介、栏目入口和读者导览。
- 设定专题、能力系统、文明、经济、航行、AI、信息治理等稳定页面。
- 概念、组织、物品、地点、角色等条目页面。
- 时间线事件、事故节点和制度演化节点。
- 世界内报道、档案、任务简报、审计记录、课堂讲义和研究笔记。
- 小说片段、作品展示稿、口述回忆、书信、群像和冷峻纪实文本。
- Matrees 提案说明、编辑摘要和结构清理备注。

## 二、基础工作流

1. 打开 Matrees 页面，确认目标世界和页面可访问。
2. 识别正确的编辑或提案入口。世界概念页面通常使用 `世界概念` 附近的分支/提案图标。
3. 读取本地权威来源时，中文 Markdown 必须使用 UTF-8 编码。
4. 粘贴前把 Markdown 转成富 HTML。不要在 Matrees 富文本编辑器里直接粘贴未转换的 Markdown，除非页面明确支持 Markdown 渲染。
5. 粘贴时尽量同时提供 `text/html` 和 `text/plain` 剪贴板内容，或用其他方式让 TipTap/ProseMirror 收到语义节点。
6. 提交提案时写短说明，说明是结构清理、富文本整理或公开阅读改写。
7. 刷新页面后检查实际渲染结果，确认标题、列表、表格是真实富文本节点，不是字面 `##`、`- ` 或管道符表格。

## 三、受保护页面

`浏览须知` 是 Matrees 同步中的受保护页面。

除非用户在当前回合明确要求编辑 `浏览须知`，否则不得删除、改名、覆盖、替换或向它提交提案。同步任务提到保留 `浏览须知` 时，只检查它在同步前后仍存在，不修改内容。

## 四、Markdown 转富文本规则

优先使用技能脚本进行可重复转换：

```powershell
python "<skill>/scripts/md_to_matrees_html.py" "source.md" --html out.html --text out.txt
```

转换目标：

- `#`、`##`、`###` 转为 `<h1>`、`<h2>`、`<h3>`。
- Markdown 项目符号列表转为 `<ul><li>`。
- Markdown 表格转为 `<table><thead><tbody>`。
- 普通段落转为 `<p>`。
- 本地仓库链接和代码包裹的本地路径应改成清洁的公开文本，或在网站正文中省略。

表格可读时优先保留为真实表格；如果 Matrees 阅读区域太窄，表格过宽，则改为分段说明或小型对照块。

## 五、网站正文风格

网站正文风格适合公开设定页、导览页、概念页、索引页和 Matrees 提案说明。改写前先选一种主导风格，除非原文明确包含不同内容块。

| 风格 | 用途 | 写法要点 |
| --- | --- | --- |
| 导读说明型 | 世界概览、分类入口、读者入口。 | 简洁可读，告诉读者页面覆盖什么；避免源文件语言、工作流痕迹和过深维护细节。 |
| 正式设定型 | 科技、文明、经济、AI、航行、能力系统等稳定专题。 | 中立第三人称，优先写定义、边界、代价、制度和后果。 |
| 条目档案型 | 概念、组织、物品、地点、角色等索引式页面。 | 段落紧凑，先写身份和功能，再写来源、现状、限制和相关主题。 |
| 时间线事件型 | 时间线与事件页面。 | 交代时间、地点、参与方、原因、过程、结果和长期影响，不写成小说场景。 |
| 问答释疑型 | FAQ、概念澄清、误解修正。 | 用直接问答说明设定限制和社会后果，适合新读者。 |
| 规则条款型 | 公共规则、服务条款、许可边界、制度说明。 | 使用分条、编号或清晰规则块，写清触发条件和后果。 |
| 操作流程型 | 流程说明、申请路径、任务步骤、公开程序。 | 按前置条件、执行、复核、异常分支和最终状态推进。 |
| 案例复盘型 | 事故后总结、制度案例、治理经验、标准修正。 | 写清发生了什么、为什么发生、后来改了什么，以及案例说明什么。 |
| 对照矩阵型 | 文明差异、制度对比、技术路线比较。 | 用小型表格加短解释；不要让矩阵替代完整专题。 |
| 发布索引型 | 专题目录、作品索引、阅读路径、页面导航。 | 说明先读什么、各入口用途是什么、条目之间如何关联。 |
| 世界内记录型 | 世界内简报、档案、交班、公开记录。 | 只在需要世界内文本时使用，让流程、授权、审计和工作习惯自然显现。 |
| 提案说明型 | Matrees 提案备注、编辑说明。 | 短、操作性强，只说明结构清理、来源保留和公开阅读调整。 |

## 六、世界内文档风格

世界内文档风格适合写成以太纪元内部机构或个人真实产出的文本。使用时应明确发文机构、时间、地点、记录对象和工作流程。不要像外部百科一样解释“权限很重要”“维护很重要”，而要让灯号、签名、排班、停机、冷却、复检、改签和审计记录体现这些共识。

| 风格 | 用途 | 写法要点 |
| --- | --- | --- |
| 新闻通讯型 | 地方新闻、工程通讯、公共服务报道、殖民地简讯。 | 面向普通居民，先写发生了什么、影响谁、下一步怎么安排。 |
| 事故报告型 | 事故档案、初步调查、责任复核。 | 包含编号、类型、时间线、影响、原因、处置和待补项。 |
| 任务简报型 | 行动前说明、工程任务、救援、巡检和维护部署。 | 写目标、窗口、分工、限制、风险和完成标准。 |
| 审计记录型 | 账本摘录、合同中止、索赔复核、维护责任链。 | 使用时间戳、状态变化、审计说明和挂起事项。 |
| 研究笔记型 | 科研观察、样本复核、深层以太记录、实验日志。 | 保留不确定性、修正过程和研究者判断，避免全知解释。 |
| 课堂讲义型 | 职业学校、公共教育、训练手册、安全课。 | 用世界内学习者能懂的实践规则、常见错误和训练标准说明。 |

## 七、叙事文本风格

叙事文本适合 Matrees 作品、小说片段、世界内叙事和展示稿。不要把稳定设定页强行小说化，除非用户明确要求叙事呈现。

| 风格 | 用途 | 写法要点 |
| --- | --- | --- |
| 人物近景型 | 单一角色感知、身体状态、工作习惯和即时压力。 | 紧贴角色注意到什么、忍受什么、决定什么。 |
| 行动现场型 | 救援、战斗、追逐、抢修、事故响应。 | 句子更短，使用倒计时、警报、具体动作和可见后果。 |
| 日常切片型 | 学校、通勤、家庭、市场、维护日和普通社会纹理。 | 让世界观从日常流程里露出，而不是解释出来。 |
| 悬疑调查型 | 遗产样本、事故复核、身份异常、审计缺口、深层污染信号。 | 从具体不一致开始，通过证据逐步显出风险。 |
| 口述回忆型 | 访谈、老兵记忆、开拓者口述、家族回忆。 | 允许个人措辞、选择性记忆和时间沉淀后的判断。 |
| 抒情史诗型 | 大历史、迁徙、古文明遗产、星海边界。 | 语言可以更有节奏和尺度，但仍保留技术限制和制度纹理。 |
| 群像交错型 | 同一事件的多角色、多地点、多机构切面。 | 转场清晰，让社会系统从并行动作中浮现。 |
| 书信通讯型 | 私人信件、远程留言、亲友沟通。 | 用关心、担忧、日常建议和地方细节显出世界。 |
| 审讯问答型 | 听证、审讯、调查访谈、责任追问。 | 用问答揭露压力、矛盾、责任链和制度冲突。 |
| 梦境回响型 | 梦境、事故后回响、冷却期幻觉、深层污染边缘体验。 | 保持暧昧，但要保留设定限制和后续记录动作。 |
| 传闻轶事型 | 地方传闻、职业老话、社区记忆。 | 让文本像居民会反复讲的话，同时留出普通程序解释。 |
| 冷峻纪实型 | 撤离、死亡或近失事件、克制纪实片段。 | 用事实、时间、数字和普通后果推进，少抒情。 |

## 八、浏览与编辑检查

在 Matrees 编辑器中，如果粘贴 HTML 后列表、表格被拍平成普通段落，应改用编辑器工具栏：

1. 先粘贴结构清晰的正文，并用标题控件建立 `H1` 到 `H4`。
2. 短枚举用无序或有序列表按钮，不把 `•` 当成真实列表。
3. 小型比较数据用表格工具填入，过宽表格改为分段说明。
4. 提交前确认编辑器 DOM 中存在真实 `h1`、`h2`、`ul`、`li`、`table` 等节点，或明确记录已改用非表格布局。

提交后刷新页面并检查：

- 公共页面显示新内容。
- 标题没有残留字面 Markdown 前缀。
- 列表不是以 `-` 开头的普通段落。
- 表格是真实表格，或已经改成有意选择的散文结构。
- 网站正文不出现本地路径，如 `../03_专题设定/...`。
