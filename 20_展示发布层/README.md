# 展示发布层

本目录是“以太纪元”面向 Matrees 和后续独立建站的中间发布层。

它不替代 `10_世界观正文/00_总纲/`、`10_世界观正文/10_专题设定/`、`90_协作工作台/10_设定推导与蓝图/` 等权威写作目录，而是把本地资料转换为网站更容易识别的内容对象：背景、设定、时间线、地图、作品和关系图谱。

## 目录分工

- `00_世界背景/`：对应 Matrees 的“背景”，用于世界基本信息、世界概念和世界历法。
- `10_设定/`：对应 Matrees 的“设定”，按网站设定集树维护。`正式专题设定/` 对应正式正文专题；`角色/`、`物品/`、`组织/`、`地理/`、`能力/`、`其他/` 对应 Matrees 内置系统节点和独立展示对象。
- `20_时间线与事件/`：对应 Matrees 的“时间线&事件”，每个重要历史节点或剧情节点应尽量拆成独立事件文件。
- `30_地图/`：对应 Matrees 的“地图”，用于星域、航道、空间锚、文明边界等地图素材和说明。
- `40_作品/`：对应 Matrees 的“作品”，用于小说、章节、世界内报道、展示稿和发布文本；纯风格样稿统一放入 `90_协作工作台/20_写作指南与样稿库/世界内文本样稿/`。
- `50_关系图谱/`：对应 Matrees 的“关系图谱”，用于角色、文明、组织、技术、事件之间的关系索引。
- `90_发布记录/`：记录本地权威文件与网站栏目、分类、标签、发布状态之间的映射。

## 当前 Matrees 设定树

2026-06-12 已将 Matrees“设定”栏目整理为两条非系统主线：

- `正式专题设定`：对应 `10_世界观正文/10_专题设定/`。
  - `能力系统`：对应 `10_世界观正文/10_专题设定/能力系统/`。
  - `星际航行和开拓秩序`：对应 `10_世界观正文/10_专题设定/星际航行和开拓秩序/`。
  - `文明思潮、道德风向与社会共识`：对应 `10_世界观正文/10_专题设定/文明思潮、道德风向与社会共识.md` 的拆分发布条目。
- `90_协作工作台`：对应 `90_协作工作台/`，只承载工作台和发布维护资料。
  - `设定工作台`：对应 `90_协作工作台/10_设定推导与蓝图/`。
  - `素材与导出`：对应 `90_协作工作台/30_素材与导出/`。

Matrees 内置系统分类仍保留为 `角色`、`物品`、`组织`、`地理`、`能力`、`其他`。发布时优先按上述设定树确定归属，再按网站分类选择展示分类。

本地发布层已同步为同构结构：正式专题整理稿集中放入 `10_设定/正式专题设定/`，系统分类目录只保留独立对象、公开导读和后续可拆分页面，避免 `能力` 分类与 `能力系统` 专题、`地理` 分类与 `星际航行和开拓秩序` 专题重复承载同一内容。

## 元数据规则

发布层 Markdown 文件建议保留 YAML frontmatter：

```yaml
---
title:
matrees_section:
matrees_category:
tags: []
source:
status:
---
```

- `title`：网站展示标题。
- `matrees_section`：Matrees 顶层栏目，例如背景、设定、时间线&事件、地图、作品、关系图谱。
- `matrees_category`：Matrees 设定分类，例如设定集、能力、角色、组织、地理、物品、其他；非设定栏目可留空或写对应子类。
- `tags`：网站筛选和独立建站用标签。
- `source`：本地权威来源文件。发布层内容不得脱离来源文件长期独立演化。
- `status`：`stable`、`draft`、`workbench`、`deprecated-source`、`asset-index` 等。

## 维护原则

- 新增正式专题后，应在 `90_发布记录/Matrees发布映射表.md` 登记发布位置。
- 工作台、蓝图和检查清单不放入本层正文对象；如需登记，只在发布映射中标明为工作台来源。
- 角色、事件、文明、地点等适合网站对象化展示的内容，应优先拆成独立发布文件。
- 图片素材仍以 `90_协作工作台/30_素材与导出/图片素材/` 为原始管理区；本目录只做发布引用和说明。
- 如果 Matrees 分类变化，先更新本目录映射表和 AGENTS.md 规则，再调整具体发布文件。
