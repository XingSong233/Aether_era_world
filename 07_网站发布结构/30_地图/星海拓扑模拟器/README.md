# 星海拓扑模拟器

本目录保存“以太纪元”第一版可计算地图原型。它不是正式星域定稿，而是用于验证一种地图表达方式：先把星海写成节点、航路、成本和文明视角，再由程序渲染为可交互界面。

当前界面采用沉浸式 HUD 结构：星海地图全屏铺底，左侧浮动控制台负责文明视角、船型、任务和图层切换，顶部状态栏负责全局提示，右侧航线档案负责节点详情、路径结果和成本模型。

## 本机运行

在项目根目录运行：

```powershell
python -m http.server 4173
```

然后打开：

```text
http://localhost:4173/07_%E7%BD%91%E7%AB%99%E5%8F%91%E5%B8%83%E7%BB%93%E6%9E%84/30_%E5%9C%B0%E5%9B%BE/%E6%98%9F%E6%B5%B7%E6%8B%93%E6%89%91%E6%A8%A1%E6%8B%9F%E5%99%A8/
```

如果从 `index.html` 直接双击打开，浏览器可能阻止读取 `data/starsea-topology.json`，因此推荐使用本地服务器。

## 文件分工

- `index.html`：交互界面结构。
- `styles.css`：布局、面板、画布和响应式样式。
- `app.js`：拓扑渲染、图层控制、节点选择、路径计算和成本模型。
- `starfield.js`：通过 Three.js 渲染全屏星海、以太雾场和环形扫描底层；Three.js 无法加载时会保留 Canvas 地图降级显示。
- `particle-field.js`：通过 tsParticles 渲染漂浮星尘、细光链和鼠标扰动层；加载失败时不影响路径计算。
- `data/starsea-topology.json`：星海节点、航路、文明视角、船型和任务目标数据。

## 视觉层依赖

当前版本使用本地固定的视觉层依赖：

```text
vendor/three.module.js
vendor/tsparticles.slim.bundle.min.js
```

这些依赖只负责星海底图、氛围层和粒子层，不参与路径计算。`three.module.js` 来自 Three.js `0.165.0`；`tsparticles.slim.bundle.min.js` 来自 tsParticles `3.x` slim bundle。它们固定在本目录是为了避免本机预览时受 CDN 或浏览器跨域策略影响。

## 当前模型

路径计算以节点和边为基础。节点包括星门、空间锚、救援链、殖民地、封存库、深层站、遗迹和异常信号。边包括星门通行、以太洋流、锚链航行、自由跃迁和暗区潜航。

有效通行成本由以下因素合成：

- 能源成本
- 时间成本
- 危险暴露
- 航权许可
- 复检压力
- 保险压力
- 救援缺口
- 航线容量
- 船型限制
- 文明视角权重
- 任务目标权重

同一条航线在联邦、环带、裂谷和商盟视角下会得到不同成本；同一节点在不同图层下也会呈现为交通枢纽、风险热区、文明影响区或禁限航迹的一部分。
