# 专题写作辅助与检查清单

状态：采用
定位：工作台辅助资料，不是世界观正文专题。
基准：`AGENTS.md`、`10_世界观正文/10_专题设定/`、`10_世界观正文/10_专题设定/能力系统/能力系统索引.md`

本文件集中存放各专题迁出的写作用途、模板格式、检查清单和旧口径修正说明。正式世界内设定仍以 `10_世界观正文/00_总纲/以太纪元世界观总纲.md` 和 `10_世界观正文/10_专题设定/` 下的专题正文为准。

## 一、通用分层规则

- 世界内正文写入 `10_世界观正文/10_专题设定/`：只保留设定口径、制度结构、技术边界、示例对象和文明差异。
- 写作辅助写入本文件：包括“写作用途”“写作检查清单”“模板格式”“避免写法”“新增设定时检查”等。
- 工作台、蓝图、状态表、矩阵和检查清单只能作为辅助资料，不应误写成 Matrees 或本地权威正文专题。
- 如果辅助规则改变核心设定口径，应同步更新总纲、相关专题、版本索引和 CHANGELOG。

### 后续整理提示

- 继续完善术语表、文明差异、职业模板、角色模板、剧情年表、经济体系与生产关系。
- 角色和剧情草稿先维护在 `草稿与候选/`，稳定后再整理进正式专题或展示发布层对象。
- 主线矛盾、推荐主线元素和剧情年表属于剧情工作底稿，不应放入权威总纲。

### 世界内文本与报道口吻

- 世界内报道、公开简报、事故档案和人物口述不应像外部设定说明一样显性强调技术、权限、风险、维护和复检的重要性。
- 在星海联邦语境中，接口复检、授权窗口、审计记录、降级运行、停机冷却、责任改签和维护排班属于隐性社会共识，应通过现场动作、职业流程、灯号、表单和人物习惯自然呈现。
- 如果需要让读者理解劳动过程差异，优先写“谁在排队复检、谁被系统锁进休息周期、哪一路灵械退回低功率、哪个记录被改签、哪个接口窗口刷新”，不要直接写“技术/权限/风险很重要”。

## 二、能力系统辅助

### 能力设定分流

新增能力设定时，先判断它主要改变运行机制、结构原理、架构演变、标准化体系、谱系体系、职业模板还是生产关系。若同时影响多类问题，应在主文件中写正文，并在其他相关文件中补充引用或边界说明。

### 个体能力运行与成长机制

来源：`10_世界观正文/10_专题设定/能力系统/个体能力运行与成长机制.md`

#### 灵脉能力设计模板

新增具体能力时，建议按以下字段记录，保证能力有运行逻辑、限制和后果。

| 字段 | 说明 |
| --- | --- |
| 能力名称 | 使用世界内名称，不使用临时外部比喻。 |
| 功能目标 | 说明能力要解决什么问题。 |
| 输入、处理、输出 | 输入来自环境、身体、灵械或网络；处理由哪些灵段完成；输出表现为感知、强化、控制、通讯或物理效果。 |
| 依赖灵核参数 | 说明主要依赖容量、算力、吞吐量、稳定性、亲和度或可塑性中的哪些维度。 |
| 体内灵脉结构 | 说明感应、筛选、放大、转换、接口、稳定和调控结构如何组合。 |
| 外置灵械协同 | 说明是否依赖灵械、供能、显示、运算、冷却、权限或团队网络。 |
| 资源占用 | 标明占用等级，并说明主要占用结构空间、算力线程、吞吐通道、注意力窗口、稳定冗余或接口带宽。 |
| 训练方式 | 说明如何唤醒、调频、固化、协同、压力适配或重构。 |
| 维护周期 | 说明日常检查、职业校准、战后复检或遗产隔离审查。 |
| 常见失效模式 | 说明失效时是漂移、串扰、烧蚀、反噬、倒灌、污染还是人格回响。 |
| 副作用 | 说明身体、神经、心理、社会身份或法律风险。 |
| 可升级方向 | 说明可以扩展为更高分辨率、更大范围、更快响应、更强协同或更高权限。 |
| 文明监管态度 | 说明联邦、环带、裂谷、商盟或遗产审查机构如何看待该能力。 |
| 剧情用途 | 说明它适合用于侦察、战斗、救援、事故调查、伦理冲突或角色成长。 |

#### 建立过程写作模板

描写一个角色建立新灵脉时，可以按以下字段记录。

| 字段 | 说明 |
| --- | --- |
| 建立目标 | 角色想解决什么问题，以及该目标属于哪个职业或剧情压力。 |
| 承载评估 | 灵核六维、神经承受力、既有灵脉冲突和维护条件。 |
| 结构来源 | 自然生成、训练诱导、标准铭刻、定制改写、外置协同或遗产接枝。 |
| 预演结果 | 沙箱、外置试运行、意识反馈或团队兼容测试中暴露了什么问题。 |
| 初接方式 | 低功率接入哪些灵纹、灵环、灵节或灵段，设置了哪些限流和断连。 |
| 共适应变化 | 新灵脉如何被个体调整，身体、灵核或既有灵脉又如何反向适应。 |
| 固化位置 | 灵核内铭、灵核外壳、神经并联、器官附着、体表外场或外置灵械。 |
| 适配训练 | 如何学习开关、调参、异常识别、多模块协同和压力复测。 |
| 验收标准 | 由谁验收，采用哪类标准，哪些条件下禁止使用。 |
| 失败预案 | 承载不足、反馈排异、漂移、烧蚀、倒灌或制度阻断时如何处理。 |
| 长期影响 | 对职业路线、心理状态、社会身份、维护债务和剧情冲突的影响。 |

#### 新增角色能力检查

- 角色通过什么渠道获得该灵脉，是否合法，是否需要职业、军方、医疗、商盟或遗产审查许可。
- 是否存在外部管控手段；该手段限制的是外置灵械、接口、环境、维护资格还是附加监管结构，而不是直接遥控体内灵脉本体。
- 这条灵脉是如何建立的：自然生成、训练诱导、标准铭刻、定制改写、外置协同还是遗产接枝。
- 建立前是否做过承载评估、预演测试、低功率接入和失败预案。
- 灵脉出现后是否经历共适应；共适应是顺向贴合、功能偏移、旧脉让路、旧脉排斥、错向固化还是失控共适应。
- 依赖哪些灵核参数。
- 占用哪些资源槽位。
- 是体内灵脉、外置灵械，还是二者协同。
- 是否需要接口灵脉、供能系统或团队网络。
- 有哪些副作用、维护需求和失效模式。
- 如何训练、升级、调频、断连、修复、变更或抹除。
- 是否改变角色职业路线、社会地位或心理状态。

### 灵脉结构原理与工程演变案例

来源：`10_世界观正文/10_专题设定/能力系统/灵脉结构原理与工程演变案例.md`

#### 写作用途

- 解释一条灵脉内部为什么要加入分层滤波、平衡放大环、限流段、断连段、稳相缓冲段、多相导流环或沙箱结构。
- 把外部工程类比转成世界内结构语言，但术语必须直白易懂。
- 为谱系文件补具体结构演变案例，避免只写应用场景或功能分支。
- 为角色能力、事故调查、维护记录和升级路线提供可检查的结构依据。

#### 写法边界与术语转译

灵脉结构可以借鉴工程思路，但正式专题正文不直接使用现实技术类比。外部沟通中的现实工程名词，应转译成世界内可理解的结构语言。

推荐转译口径：

| 外部沟通说法 | 世界内结构说法 |
| --- | --- |
| 对称放大 | 平衡放大环、平衡放大节点、平衡放大回路 |
| 整流桥、分路整流 | 多相导流环 |
| 纹波改善、输出抖动改善 | 稳相缓冲段、输出平滑段 |
| 直接接入、单级接入 | 直连回路 |
| 过载保护、功率限制 | 限流段 |
| 紧急切断、保护断开 | 断连段 |
| 隔离测试、封闭试运行 | 沙箱段 |

术语可以根据文明、职业或家族风格改名，但核心表达必须清楚：它解决什么问题，怎么解决，代价是什么。不要为了显得复杂而堆术语；一个结构名称如果不能让读者大致看出用途，就应换成更直接的说法。

#### 新增结构案例检查

- 是否写清早期方案的简单之处，而不是直接给出成熟结构。
- 是否说明早期方案暴露了哪些具体缺陷，例如误读、过曝、输出变形、倒灌、烧蚀或污染反冲。
- 是否说明新增结构解决了什么问题，以及带来什么代价。
- 结构名称是否直白，如直连回路、平衡放大环、平衡放大节点、平衡放大回路、分层滤波、稳相缓冲段、输出平滑段、多相导流环、限流段、断连段、备用路径、沙箱段。
- 是否避免把现实工程名词直接写进正文，或把术语写得冗长拗口。
- 是否说明衍生功能仍然继承原有主结构，而不是因为功能相似就归入同一谱系。
- 新增结构案例是否至少说明：早期方案为什么简单、暴露了什么缺陷、新增了哪些结构、改造后解决了什么问题、带来什么代价。

### 灵脉架构演变

来源：`10_世界观正文/10_专题设定/能力系统/灵脉架构演变.md`

#### 写作用途

- 解释为什么现代角色不把所有能力都刻进体内。
- 区分旧式强者、军校学生、职业者、商盟模块使用者和裂谷实验体。
- 设计角色能力卡时说明体内核心、接口层、外置模块和团队网络如何分工。
- 通过接口延迟、断连、反馈压缩误判、模块战损和维护债务制造代价。

#### 新增能力架构检查

- 哪些功能段保留在体内，哪些迁移到灵械或团队系统。
- 体内保留的通用灵脉是否承担低延迟、自主性或断连生存任务。
- 接口灵脉负责哪些识别、同步、转译、限流、断连和污染隔离。
- 外置模块承担了哪些高功率、高风险、复杂处理或维护密集结构。
- 反馈是原始直连、压缩提示、权限提示，还是团队态势摘要。
- 失效时是体内灵脉受损、接口断连、外置模块战损、权限撤销，还是污染倒灌。
- 是否误把架构分工写成灵脉谱系；若只是在说明承载位置和协作方式，应归入架构演变，而不是谱系文件。

### 灵脉与灵械标准化体系

来源：`10_世界观正文/10_专题设定/能力系统/灵脉与灵械标准化体系.md`

#### 写作用途

- 角色能力卡：说明角色采用的标准规格、定制范围、维护负担和禁用条件。
- 职业模板：区分职业标配、进阶套件、战时套件和个人改装。
- 获取冲突：通过许可等级、培训资格、非法持有、退役封存或授权密钥推动事件。
- 管控冲突：通过外置系统锁、附加监管灵脉、医疗休眠、司法封存和组织伦理差异推动事件。
- 装备冲突：通过兼容失败、认证失效、权限锁定或黑市后门推动危机。
- 剧情任务：围绕认证测试、事故调查、遗产隔离、军标走私或商盟仲裁展开。
- 技术伦理：讨论谁有资格改造灵核、谁承担定制事故责任，以及危险模块是否可商品化。

#### 新增定制能力检查

- 采用哪类标准规格。
- 通过什么许可等级获取，是否允许普通平民、职业人员、军方或遗产机构持有。
- 是否存在管控手段；管控对象是外置模块、接口、环境、维护资格还是附加结构，是否越过灵脉本体自主边界。
- 依赖哪些灵核参数和接口灵脉等级。
- 占用多少容量、算力、吞吐、稳定冗余、注意力窗口或接口带宽。
- 是否需要认证、维护、授权密钥或专业团队支持。
- 失效时是降级、断连、污染、劫持，还是伤害使用者。
- 是否触及文明法律、军用管制、商盟契约或肃正协议警戒。

### 灵脉谱系设定

来源：`10_世界观正文/10_专题设定/能力系统/灵脉谱系设定.md`

#### 旧口径修正

`v0.5.0` 中新增的“战术推演谱系、生命扫描谱系、护盾发生谱系、神经加速谱系、污染隔离谱系、共鸣通讯谱系、以太牵引谱系”更接近功能分类，不应作为成熟谱系名称使用。上述名称改归 `10_世界观正文/10_专题设定/能力系统/个体能力运行与成长机制.md` 的“灵脉功能分类参考”，只用于说明灵脉主要做什么。

后续谱系示例必须能写清原型来源、早期方案缺陷、关键工程改造、主干继承、分支偏移、衍生功能、旁支风险、断代或接枝关系。若只能说明能力用途、资源压力或职业适配，应放入能力体系或职业模板，不放入谱系文件。

#### 写作用途

- 角色能力卡：说明角色的核心灵脉来自哪条谱系，继承了什么，偏离了什么。
- 职业模板：说明职业标配灵脉的主干来源，以及允许哪些安全分支。
- 家族与文明差异：说明某个家族、军方、商盟工坊或裂谷实验室为何形成独特能力路线。
- 剧情冲突：通过谱系断代、遗产接枝、分支争夺、公开程度和维护权归属推动事件。
- 维护与事故调查：通过追溯分支来源判断故障来自原型缺陷、定制改写、接口污染还是非法旁支。

#### 新增灵脉谱系检查

- 它的原型逻辑是否清楚。
- 它是否写出了“早期粗糙方案 -> 缺陷暴露 -> 工程改造 -> 成熟主干 -> 分支衍生”的因果链。
- 功能分类和应用场景是否只是迭代压力，而不是被直接写成谱系名称。
- 它是成熟谱系、实验分支、断代谱系，还是尚未成谱系的原创设计。
- 它和既有功能分类、标准状态、定制逻辑是否混淆。
- 它的主要分支如何继承或偏离原型结构。
- 它是否说明哪些功能是主干成熟后衍生出来的，哪些旧路线被封存、限制或仅在特殊场景复启。
- 它适配哪些灵核参数、职业任务、文明制度或灵械接口。
- 它的公开程度、维护体系和失效模式是什么。
- 它是否带来同源兼容、频率冲突、接口污染、认知过载或谱系污染。

## 三、专题设定辅助

### 文明思潮、道德风向与社会共识

来源：`10_世界观正文/10_专题设定/文明思潮、道德风向与社会共识.md`

- 正式专题只保留社会评价层、内部思潮、阶层共识、跨文明争议和相关制度层，不放写作指南段落。
- 不要把思潮写成法律条文；法律决定可执行结果，思潮决定人物如何评价、辩护、羞耻、抗议或自我合理化。
- 不要让四文明只有外部冲突；每个文明内部都应有能自洽辩论的主流派、激进派和修正派。
- 不要把文明内部阶层写成单纯贫富差距；以太纪元的阶层更常由训练、维护、接口、航权、封存权、实验资格、密钥信用和救济速度构成。
- 不要把高风险行为写成只有勇敢或怯懦；应检查谁决定风险、谁承担代价、弱势主体能否拒绝、记录能否更正、救济能否抵达。
- 不得恢复已禁用口径：文明级 AI、记忆编辑、复活、时间工程、小型传送、惯性控制、近乎免费能源、无成本遗产外挂、精准出生职业定制。
- 正式专题表达中避免出现“写作时”“适合写”“叙事功能”“剧情用途”“使用边界”等写作指南词；需要保留的规则迁入本文件。

### 联邦思潮状态与制度底线

来源：`10_世界观正文/10_专题设定/联邦思潮状态与制度底线.md`

- 不要为联邦文明设定单一、具体、决定性的起源事件或起源战争。
- 不要使用已撤回的具体历史专名、霸权体系名、战争名、遗迹名或战后公约名来解释联邦现状。
- 不要把联邦写成几个天然互补的种族自动合并，也不要把主体族群或泰拉人族亚种族差异写成文明短板拼装、固定职业命运或天然等级。
- 不要把联邦写成没有资源争抢、没有阶层分化、没有制度拖延的理想国。
- 不要让“遗产共享”变成无成本公开所有古文明技术；高危样本、污染风险、军事用途和主体稳定风险仍需隔离审查。
- 不要把“开拓”写成插旗即主权；联邦开拓强调测绘、作业、锚站、基础服务、居民权利、救援和自治承认。
- 不要把联邦公共审计写成绝对正确的无人裁判；法院、仲裁、专业委员会、舰队审计和地方机构都可能冲突、延误或犯错。

- 写经济冲突时，优先检查能源、航权、接口、维护、许可、保险和仲裁，而不是只写金钱差距。
- 写基础服务时，区分生存保障和机会平等；基础生活稳定不等于无阶层。
- 写能源额度时，保留基础设施、安全许可和场域稳定限制。

- 写智能合约和共鸣审计账本时，保留司法、仲裁、伦理审查、军事管制和文明监管入口。
- 写 AI、数字人格和 L5 多线程劳动时，必须处理人格稳定周期、载体维护和权利地位。
- 写高危样本和遗产对象时，不把“持有实物”直接等同于合法所有权。
- 写职业贫困时，优先表现训练名额、接口接入、维护资格、事故记录、迁徙资格和法律救济不足。
- 写产业结构时，正文标题优先使用生态资源、工业、能源与制造、流通服务与治理等直观名称；第一产业、第二产业、第三产业保留为维护分类骨架，不单独替代正文表达。

### 基础军事力量架构

来源：`10_世界观正文/10_专题设定/基础军事力量架构.md`

- 不要把通用架构写成所有文明军队使用同一名称、同一军衔、同一编制。
- 不要让单兵武器、机甲或 AI 取代舰队、航权、后勤和维护体系。
- 不要把星门或自由跃迁写成无成本传送，也不要让小队随意绕过航权、安检、能源和复检。
- 不要把体内灵脉写成可以被军方、企业或敌人实时遥控；管控对象优先写外置灵械、舰载系统、授权密钥和任务权限。
- 不要把高过载载人机甲写成无视惯性。超过载人承受窗口的高速任务交给无人系统和导弹。
- 不要把护盾写成无成本、不可见、不可探测、单向允许己方攻击穿出的万能防御。护盾应同时阻碍攻防双方，并受面积、强度、方向、距离、能源、散热、发生器阵列和维护窗口限制。
- 不要把遗产武装、深层以太武器和认知压制写成常规便利武器。它们必须带来隔离、污染、伦理、肃正协议或跨文明争议。
- 不要忽略战后复检、医疗救济、维修记录、保险承保、审计复盘和退役降级。这些是本世界军事力量可信度的一部分。

### 生产关系与职业结构

来源：`10_世界观正文/10_专题设定/生产关系与职业结构.md`

#### 生产资料重心的现实直观对照

下表只用于帮助读者获得直观感觉，不是世界内术语替换。现实对照只能说明“社会位置相近”，不能说明技术原理相同。

| 生产资料 | 现实直观对照 | 为什么算生产资料 |
| --- | --- | --- |
| 以太场与能源基础设施 | 电网、燃气管网、数据中心供电、危化工厂安全系统、洁净厂房环境控制。 | 它决定城市、舰船、工厂、殖民地和高危工区能否持续开工；掌握供能、屏蔽、清洁、熔断和复检能力，就掌握了大量生产活动的启动条件。 |
| 接口与权限系统 | 工业控制权限、API 密钥、职业执照、设备安全认证、特种作业许可。 | 它不直接生产物品，但决定谁能合法接入灵械、职业套件、军用设备和共鸣系统；没有权限和维护记录，劳动者有技能也不能进入正式生产链。 |
| 高端制造设备 | 半导体晶圆厂、光刻机、数控机床群、自动化工厂、精密化工产线。 | 它把设计和材料转化为可复制产品；当普通物品稀缺性下降后，真正稀缺的是装配能力、材料配方、能源额度和设备授权。 |
| 星门、航道与空间锚 | 港口、机场、铁路枢纽、运河、导航卫星、空管系统。 | 它决定货物、人口、舰队和信息能否低风险流动；控制节点、窗口、航权和救援能力，就能影响市场、殖民地开发和军事投送。 |
| 灵械与职业套件 | 挖掘机、医疗设备、专业软件、工业机器人、外骨骼和成套作业工具。 | 它把个人能力放大成稳定职业产出；劳动者拥有技能，但套件的维护、授权、保险和规格常由机构控制。 |
| 数据、蓝图与风险模型 | 专利、工艺配方、CAD 图纸、航海图、保险精算模型、事故数据库。 | 它决定能造什么、怎么造、能不能投保、能不能通行、事故后如何追责；解释权和模型更新权会直接影响产业准入。 |
| 生命与身体维护能力 | 医院、职业病防治、康复体系、器官移植、辅助生殖和高危岗位体检。 | 它维护劳动者的身体连续性和职业能力；在再生医学成熟后，真正稀缺的是灵核治疗许可、复健资源、职业复检和高危任务前后的医疗承保。 |
| 智能主体与载体维护能力 | 没有完全对应物，可部分参照关键专家劳动合同、服务器托管、数据中心运维和长期医疗维护。 | 高等级 AI 和数字人格不是生产工具本身，而是有权利的劳动主体；真正的生产资料是载体维护、人工灵核维护、登记系统、合同结构和人格稳定评估能力。 |
| 金融合约与审计基础设施 | 银行清算系统、支付网络、产权登记、保险体系、公证与法院执行系统。 | 它让资产能被租赁、抵押、赔付、冻结、解冻和跨区域结算；生产活动依赖资金流、风险分摊、产权确认和纠错救济。 |
| 恒星生态与轨道级巨构 | 大坝、核电站、港口群、城市电网、卫星星座、超大型船坞。 | 现实没有同规模对象；它算生产资料，是因为它提供居住、重力、生态、防护、船坞、星港和能源配额，整个区域的生产生活都建立在其长期运维上。 |
| 生育与人口工程 | 辅助生殖、产前筛查、公共卫生、人口政策、殖民地移民规划。 | 它不是把个体当作物品，而是维护人口再生产、跨种族兼容、环境适应和殖民地劳动力补充的制度能力；谁能获得高端服务，会影响家庭、身份和长期机会。 |
| 高危样本与封存库 | 病原体样本库、核材料库、危废封存、矿权样本、战略科研材料库。 | 它既是研究对象也是潜在技术来源；掌握样本的隔离、转译、封存、复检和研究许可，就掌握了新技术路线和灾害风险的入口。 |

新增职业时，建议先在生产关系专题中判断它属于哪种生产关系节点，再进入职业模板库填写能力与训练细节：

- 它控制或维护哪类生产资料：能源、接口、航权、制造设备、医疗资源、数据模型、样本、巨构还是合约。
- 它属于生态资源、工业、能源与制造、流通服务与治理中的哪一类；维护分类上归入第一产业、第二产业、第三产业、跨产业岗位、高危岗位还是灰色岗位。
- 它依赖哪些许可、维护记录、保险、授权密钥、职业信用或文明身份。
- 它失败时造成个人事故、职业事故、城市事故、舰队事故、样本污染还是文明级风险预警。
- 它在四文明中的合法性、地位和救济渠道是否不同。

生产关系写作检查：

- 不要把职业写成单个角色的技能栏；先说明它在生产链、维护链、授权链和事故责任中的位置。
- 不要把自动化写成劳动完全消失；应说明哪些劳动被压缩，哪些判断、维护、认证和追责岗位被强化。
- 不要把灵核六维写成完整劳动能力评级；联邦尤其不能把灵核六维当作唯一身份或职业根基。
- 不要把灵械租赁写成纯粹便利服务；同时检查密钥冻结、维护断供、债务和救济权。
- 不要把高危样本、遗产蓝图或军标模块写成普通商品；先确认隔离、转译、封存、承保和文明监管。
- 不要把 AI、数字人格或 L5 多线程写成无责任工具；必须处理人格稳定、载体维护、权利地位和任务审计。
- 不要把胚胎设计写成稳定能力职业预设；它可以处理疾病、体质、环境适应、跨种族兼容和概率性灵核发育干预，但必须区分医疗性修复、适应性干预和高风险强化实验，并保留不稳定、赔付责任和文明监管。
- 不要让某个文明制度成为唯一正确答案；联邦、环带、裂谷和商盟都应有优势、代价和内部矛盾。

### 深层以太设定

来源：`10_世界观正文/10_专题设定/深层以太设定.md`

深层以太适合承担以下叙事功能：

- 推动远征：暗区航道、星门异常和殖民地失联迫使舰队进入未知区域。
- 放大代价：角色能力越强，越可能接触更高强度的信号和回响。
- 制造伦理冲突：同一份样本在联邦是科研资料，在环带是封存对象，在裂谷是进化钥匙，在商盟是高价值资产。
- 连接古文明：失落星庭、肃正协议、大寂灭和先驱者蓝图都可通过深层以太留下可疑线索。
- 塑造主角成长：沧月心悚的优势应体现为兼容、推演、隔离判断和多系统协同，而不是无代价承受深层以太。

涉及深层以太时，至少检查：

- 这次探查的文明组织目的是什么：预警、航行、遗产、资源、救援还是政治争夺。
- 个体接触强度属于边缘扰动、受控接入还是直接暴露。
- 使用了哪些隔离灵械、接口灵脉、撤离阈值和任务后复检。
- 风险落在哪些层面：灵核过载、感知污染、接口倒灌、人格回响、时间感漂移还是神经副作用。
- 样本和数据由谁拥有，谁有权继续研究，谁承担事故责任。
- 能力提升是否伴随新的维护成本、审查压力或心理代价。

### 航行自由跃迁与开拓秩序

来源：`10_世界观正文/10_专题设定/星际航行和开拓秩序/航行自由跃迁与开拓秩序.md`

- 写自由跃迁时，先确认锚定解、跃迁窗口、高相干能源额度、舰船适航、航迹记录、复检和目的地接纳。
- 不要把自由跃迁写成任意传送、完美隐身、无限连续短跳或低成本全民工具。
- 写星门与自由跃迁时，先区分固定设施长期维护的可重复通道，和星舰短时生成的一次性以太航迹；不要把自由跃迁写成便携星门，也不要把现代文明写成能从零制造古文明级主干星门网络。
- 写星际交通时，区分星门通行、以太洋流航行、锚链航行、自由跃迁和暗区潜航。
- 写低成本星际航行时，保留排期、补贴、检疫、航权、保险和目的地资格。
- 写边境开拓时，不要让插旗、发现或自动殖民种子直接产生完整主权。
- 写殖民地时，检查基础服务权、能源、医疗、教育、撤离、居民自治和事故责任。
- 写航运商业时，优先检查航权配额、星门窗口、空间锚使用权、泊位、检疫、救援承诺和保险，而不是只写钱。
- 写暗区或遗产航行时，必须联动深层以太、遗产隔离、污染熔断、人格回响和肃正协议预警。
- 写四文明制度时，让联邦、环带、裂谷和商盟都保留优势、代价和内部矛盾。

### 载人机甲与高速作战

来源：`10_世界观正文/10_专题设定/载人机甲与高速作战.md`

涉及高速载人机甲时，至少检查：

- 是否仍明确惯性控制技术不存在。
- 是否说明使用的是惯性承载、承压灵脉、承载舱和医疗维护，而不是惯性消除。
- 是否区分载人机甲、无人僚机、导弹、浮游武器和舰载火控的任务边界。
- 是否写清加速度窗口、持续时间、机师有效操作能力和战后复检。
- 是否说明机甲如何抵达战场近域，而不是让机甲自行跨星系高速航行。
- 是否保留能源额度、推进剂、热管理、接口带宽、注意力窗口和事故责任。
- 若数值超过 300g，是否给出对应的身体、灵核、神经、人格或制度代价。
- 若属于商盟、裂谷或黑市路线，是否说明维护链、保险链或实验担保为何允许它存在。

避免写法：

- 把高过载机甲写成已经恢复惯性控制技术。
- 让机师在 500g 以上长时间精细操作且没有副作用。
- 让载人机甲替代所有舰队、导弹、无人机和星际交通系统。
- 只写机体性能，不写机师承载、医疗复检、维护记录和事故责任。

### 术语表

来源：`10_世界观正文/10_专题设定/术语表.md`

- 若术语用于能力运行，优先引用 `10_世界观正文/10_专题设定/能力系统/个体能力运行与成长机制.md` 的口径。
- 若术语用于接口、许可、商品化、军用化或维护认证，优先引用 `10_世界观正文/10_专题设定/能力系统/灵脉与灵械标准化体系.md` 的口径。
- 若术语用于原型继承、分支演化、断代复原或遗产接枝，优先引用 `10_世界观正文/10_专题设定/能力系统/灵脉谱系设定.md` 的口径。
- 若术语用于深层以太探查、样本、回响、组织化风险或文明争夺，优先引用 `10_世界观正文/10_专题设定/深层以太设定.md` 的口径。
- 不要把功能分类写成谱系名称。感知、推演、防护、通讯、牵引等只说明“做什么”，谱系必须说明“从哪里来、如何继承、如何分化”。
- 不要把标准规格写成能力强弱等级。规格说明的是适用对象、接口、功率、维护、权限和失效模式。
- 不要把遗产隔离写成万能安全许可。它只能降低未知结构的接入风险，不能消除污染、人格回响、肃正协议警戒或维护责任。
- 不要用多个近义词替代同一概念。需要表达“接口交换上限”时使用“接口带宽”；需要表达“认知反馈上限”时使用“注意力窗口”；需要表达“工程化参数边界”时使用“标准规格”。
