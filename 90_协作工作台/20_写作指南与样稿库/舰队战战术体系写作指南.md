# 舰队战战术体系写作指南

状态：workbench
定位：写作辅助资料，不是世界观正文专题。
基准：`10_世界观正文/10_专题设定/基础军事力量架构.md`、`10_世界观正文/10_专题设定/载人机甲与高速作战.md`、`90_协作工作台/20_写作指南与样稿库/专题写作辅助与检查清单.md`

本文件用于辅助后续扩写星际舰队战、机甲协同、小型舰艇存在价值、武器破坏力平衡和四文明战术差异。这里整理的是战术写作方法，不直接新增世界内正式设定。若后续某条内容稳定采用，应整理进 `10_世界观正文/10_专题设定/基础军事力量架构.md` 或相关正式专题。

## 一、核心判断

舰队战不应写成单纯比较射程、速度、装甲和破坏力。更适合本世界观的战术核心是：不同平台围绕射击窗口、护盾窗口、锁定窗口、机动窗口、通信窗口、补给窗口和授权窗口进行资源管理。

主力舰、小型舰艇、母舰、机甲、无人僚机、导弹和固定节点的差异，不在于谁能直接替代谁，而在于它们分别占用、制造、关闭或保护哪一类窗口。

一句话口径：

> 星际舰队战不是谁的炮更大，而是谁能让敌方在错误时间、错误方向和错误目标上消耗关键资源。

## 二、现实海战参考的转译边界

二战美、日海军可作为战术组织参考，但不能直接把现实舰种、吨位、航速、射程或编制搬入以太纪元。

舰艇长度比例可以借用现实海战的保守直觉：驱逐舰与主力舰通常不是十倍、二十倍长度差，而是数倍级差。以太纪元不按现实舰艇尺寸写，但若三十米级机甲、千米上下的护航舰和主力舰需要处在同一战术生态里，常规主力舰就不宜默认写成六千米级；六千米及以上更适合作为远征旗舰、移动船坞、移动军港或极少数设施化平台的量级。

太空舰队战的最低独立有人作战平台是机甲。它相当于舰队尺度中的单兵，但因为机师和维护链具有精英性质，战术和制度地位可以适当高于普通单兵。当前采用三十米级机甲与常规主力舰约一百倍长度差，即主力舰中位尺度约三千米；不要把机甲抬成小型舰艇，也不要把常规主力舰重新放大到让机甲在工程直觉上完全失去窗口威胁意义。

可借用的抽象包括：

| 现实参考 | 可转译抽象 | 不应直接搬用 |
| --- | --- | --- |
| 美国航母特混舰队 | 母舰后置、舰载单位前出、护卫层动态保护核心平台。 | 现实航母吨位、舰载机数量、太平洋战场距离。 |
| 美国雷达与战斗信息中心 | 传感、通信、火控、护盾开窗和授权记录的统一调度。 | 把雷达等同于本世界唯一侦察方式。 |
| 美国分层防空 | 外层截击、中层拦截、近层点防御、核心平台保护。 | 只写防空炮墙，不写护盾、共鸣干扰和识别责任。 |
| 日本夜战鱼雷突击 | 隐蔽接近、短促暴露、远程齐射、打完脱离。 | 把小舰写成靠正面防护硬扛主力舰。 |
| 日本航母第一击 | 多平台集中投送，抢在敌方完成组织前制造战术失衡。 | 把第一击写成无视侦察、回收、维护和补给的万能先手。 |

现实参考只用于回答：不同攻击方式的平台如何分层、如何接敌、如何撤离、如何互相保护。

## 三、舰队位置变化的基本阶段

### 1. 搜索与态势建立

舰队核心保持在敌方主力火力和突击单位有效范围之外，侦察单位、无人哨、小型舰艇、潜伏节点或前出机甲向外展开。

写作重点：

- 核心平台相对集中，侦察层向外张开。
- 前出单位不应无限远离支援链。
- 发现异常不等于可信识别，更不等于已经获得开火授权。

### 2. 出击与投送窗口

母舰、远征旗舰或机甲母舰需要放飞、弹射、回收、整备、补能和医疗复检。这个阶段类似现实航母的飞行甲板节奏，会暂时改变舰队航向、姿态、护盾开口和火控配置。

写作重点：

- 母舰不是随时无限投放机甲或无人群。
- 出击窗口会削弱某些方向的防护或火力。
- 护卫舰和主力舰要补偿母舰在出击/回收时的脆弱窗口。

### 3. 远程打击与母舰后置

舰载机甲、无人僚机、导弹、突防单元或远程弹药前出，母舰本体后置。理想位置关系是让己方效果投送触达敌人，而敌方主力火力难以直接威胁己方核心平台。

可理解为有限意义上的“放风筝”，但不要写成整支舰队简单抱团后退。真正的战术动作是：打击力量前出，母舰保持距离，护卫层根据威胁方向变形。

### 4. 防空、反导与反突击

敌方攻击波来袭时，舰队从航行阵型转为防御阵型。战斗不只是拦截弹药，还包括识别真假目标、保护通信链、同步护盾开窗、避免误伤己方前出单位。

写作重点：

- 防御层向威胁方向加厚。
- 护盾会阻碍己方火力、传感和回收通道。
- 近防、电子战、无人拦截和小型舰艇应优先处理小目标，避免主力舰浪费主炮窗口。

### 5. 敌方主力逼近

若敌方主力舰、突击舰群或高威胁平台逼近，母舰优先拉开距离，护卫舰、主力舰、干扰舰和小型舰艇在威胁方向上形成迟滞层。

写作重点：

- 母舰避免进入敌方主炮或近距突击有效区。
- 护卫层不是静止围墙，而是动态屏障。
- 小型舰艇可以横在威胁轴上，但必须保留撤离路线和支援链。

### 6. 恢复、撤离与战损降级

一次攻击或防御后，舰队需要重新整理护盾、冷却、补弹、维修、医疗复检、机甲回收、无人群断连审计和目标再识别。

写作重点：

- 战斗不是连续满功率输出。
- 谁恢复得快，谁就能抢到下一轮窗口。
- 战损降级比瞬间爆炸更常见，也更能体现体系战。

## 四、位置控制规则

后续写舰队战时，可优先检查这些位置规则：

| 规则 | 含义 |
| --- | --- |
| 核心平台后置 | 母舰、补给舰、维修船坞舰和医疗撤离舰避免暴露在敌方主力火力内。 |
| 打击单位前出 | 机甲、无人僚机、导弹和小型突击舰承担接触风险，但不能脱离支援链。 |
| 护卫层动态变形 | 护航舰、巡洋级平台、主力舰和电子战舰根据威胁方向补位。 |
| 火力互相覆盖 | 小舰不能为了突击离开所有己方点防御、电子战和回收通道。 |
| 护盾几何优先 | 护盾方向、面积、距离和强度决定哪些方向能防、哪些方向能打。 |
| 不追求单舰全能 | 平台通过舰队链路补齐短板，脱链后必须降级。 |

## 五、资源窗口清单

舰队战术写作中，建议把“资源”写成可被观察、争夺和误用的窗口。

| 资源窗口 | 可写成的战术问题 |
| --- | --- |
| 主炮装填/充能 | 是否值得用一次主炮机会打小型目标。 |
| 护盾展开/关闭/转向 | 开火是否需要暴露舰体、弱侧或回收通道。 |
| 火控锁定 | 高速小目标是否占用过多精密火控链。 |
| 可信识别 | 看见目标后是否能确认身份、授权开火并承担误击责任。 |
| 通信带宽 | 前出单位是否仍能获得目标更新、撤离指令和黑匣同步。 |
| 能源与散热 | 高强度护盾、主炮、跃迁、医疗和共鸣通信是否互相挤占。 |
| 姿态与射界 | 为了攻击某目标是否需要转向，导致暴露其他方向。 |
| 出击/回收节奏 | 机甲、无人群和突入舱是否能及时回收、补给和复检。 |
| 指挥注意力 | 小目标、假目标和突发威胁是否压缩指挥链判断窗口。 |
| 维护与补给 | 一轮战斗后能否恢复到下一轮可用状态。 |

## 六、小型舰艇的存在价值

小型舰艇、护航舰、突击舰和前出节点的价值不是正面承受主力舰火力，而是扰动敌方资源管理。

它们可以承担：

- 前出警戒和异常发现。
- 局部可信识别和目标标定。
- 反导、反无人、反机甲和救援。
- 电子战、共鸣干扰和假目标投放。
- 近中程导弹、鱼雷或突防单元齐射。
- 诱使敌方主力舰转火、开盾、关盾、转向或浪费主炮窗口。
- 迫使敌方护卫舰离开原本阵位。

小型舰艇的核心战术价值：

> 它不一定要击毁主力舰，只要让主力舰为了处理它而错过更高价值窗口，就已经完成任务。

## 七、小型舰艇的生存规则

小型舰艇不靠厚装甲或主力级护盾生存。它们靠“不被稳定锁定、不被单独吃掉、完成任务后及时脱离”生存。

可用生存手段：

| 手段 | 本质 |
| --- | --- |
| 低可探测截面 | 延迟敌方可信识别和火控闭环。 |
| 高机动和短时变轨 | 破坏敌方射击解，缩短暴露时间。 |
| 电子战和假目标 | 迫使敌方浪费火控、拦截弹和指挥注意力。 |
| 护盾墙边缘/残骸/设施遮蔽 | 借复杂几何接近，避免进入主力舰稳定射界。 |
| 编队互相覆盖 | 避免单舰被敌方护卫层吞掉。 |
| 己方支援半径 | 保持在己方点防御、主力舰火控、母舰回收或中继覆盖内。 |
| 打完就撤 | 发射、标定、干扰或救援后立即脱离，不恋战。 |

避免写法：

- 小型舰艇在敌方主力舰正面火力区长时间盘旋。
- 小型舰艇靠同级护盾硬抗主力舰主炮。
- 小型舰艇脱离母舰、主力舰、空间锚或共鸣中继后仍拥有同等远距识别和制导能力。
- 小型舰艇单独完成侦察、识别、授权、制导、突击、撤离和战后审计全部闭环。

## 八、主力舰为什么不总是直接击毁小舰

即使主力舰有较大概率一炮击毁小型舰艇，也必须计算机会成本。

主力舰对小型目标开火可能付出：

- 浪费一次主炮装填或充能周期。
- 打开、关闭或偏转护盾，暴露给敌方主力舰。
- 占用高精度火控链。
- 为射界转向，暴露弱侧或破坏队形。
- 增加散热和能源压力。
- 错过压制敌方主力舰、母舰或固定节点的窗口。

因此，主力舰面对小舰时常有多个选择：

1. 交给近防、护航舰、无人拦截或机甲处理。
2. 用电子战和假目标拖延。
3. 调整护盾几何，拒绝小舰进入关键方向。
4. 迫使小舰离开支援链，再由护卫层处理。
5. 只有在小舰威胁足够高或敌方主力窗口不成立时，才使用高价值火力。

## 九、机甲作战的战术定位

机甲不应写成小型主力舰，也不应写成无条件反舰王牌。它更适合承担近域窗口任务。

适合机甲的任务：

- 护盾节点压制和局部开口维持。
- 星舰外壳、空间锚、船坞、残骸带和设施近域作业。
- 传感器、散热板、外置天线、武器节点和维护接口破坏。
- 目标标定、火控中继和敌方护盾薄弱方向确认。
- 登舰、救援、样本回收和低附带损伤夺控。
- 反小型单位、反突入舱和近距护卫。

限制应同时写出：

- 母舰或设施投送。
- 作战半径。
- 通信和中继依赖。
- 能源、推进剂和热管理。
- 机师注意力窗口、接口带宽和战后复检。
- 回收、维修和医疗周期。

机甲威胁大型舰时，最好需要条件：敌方护盾被牵制、传感器被压制、目标转向受限、护卫层被调走、战场存在残骸或设施遮蔽、己方主力舰提供火力窗口。

三十米级机甲威胁主力舰时，优先写成节点威胁和窗口威胁：破坏护盾发生器、散热板、传感阵列、通信天线、点防御盲区、维护接口、武器开窗机构或战损裂口。不要写成机甲用自身持续输出正面压过主力舰总体能源、装甲和护盾。

## 十、武器破坏力平衡建议

武器平衡不只看“能不能击毁目标”，还要看“击毁目标需要牺牲哪些窗口”。

| 武器/平台 | 应强调的代价 |
| --- | --- |
| 主炮/重型能量炮 | 装填、充能、散热、护盾开窗、射界和政治/审计责任。 |
| 中型炮/导弹 | 弹药、制导链、拦截风险、误击责任和补给压力。 |
| 近防武器 | 火控通道、弹药消耗、射界盲区和对己方小单位的误伤风险。 |
| 机甲武装 | 近距接敌、机师承载、接口带宽、维护和回收窗口。 |
| 无人僚机/浮游武器 | 断连、劫持、识别错误、黑匣记录和战后责任。 |
| 护盾 | 能源、散热、方向、面积、距离、己方火力阻隔和发生器节点暴露。 |
| 电子/共鸣战 | 可信识别下降、误伤风险、授权争议、污染过滤和己方通信压力。 |

## 十一、四文明战术扩写提示

四文明都可以使用舰队、母舰、小型舰艇、机甲、无人系统和固定节点，但战术偏好应从制度和技术路线推导。

| 文明圈 | 可优先扩写的战术倾向 |
| --- | --- |
| 星海联邦 | 标准化舰队协同、救援链、审计链、机甲现场授权、护卫层补位和战损降级。 |
| 永恒环带 | 阈值防御、护盾几何、封锁、隔离、撤离通道保护和长期设施防卫。 |
| 深渊裂谷 | 高风险突防、异常环境适应、实验机甲、污染区作战和非标准窗口交换。 |
| 商盟 | 护航合同、模块化火力、维护剥夺、保险约束、雇佣舰队和可结算风险转移。 |

扩写时不要让四文明只是“美式体系战”和“日式突击战”的换皮。现实海军只提供战术结构参考；文明差异必须落到授权、维护、风险承担、救援义务、合约、封存、实验伦理和社会评价。

## 十二、舰队战写作检查清单

写一段舰队战前，至少检查：

- 本段争夺的是哪个窗口：射击、护盾、识别、通信、机动、出击/回收、补给还是授权。
- 核心平台是否暴露在敌方主力火力或突击单位有效范围内；如果暴露，原因和代价是什么。
- 前出小型舰艇、机甲或无人单位是否仍在支援链内。
- 护盾是否同时影响敌我双方；是否写出开窗、转向、关闭或局部薄弱。
- 主力舰为什么不直接处理所有小目标。
- 小型单位完成任务后如何撤离、降级、断连或被回收。
- 是否区分异常发现、可信识别、火控/制导闭环、效果投送和通信接入。
- 是否保留维护、冷却、补弹、医疗复检、黑匣记录和战后审计。
- 是否避免把机甲写成远距独立反舰平台。
- 是否让战术差异来自文明制度和技术路线，而不是单纯性格或审美。

## 十三、可直接使用的战术句式

- 敌方小舰没有进入主炮决战线，却进入了护盾调度的错误位置。
- 指挥官没有下令转火，因为那会打开正面护盾的两秒窗口。
- 机甲队的任务不是击沉主力舰，而是让它无法在下一轮齐射前完成护盾转向。
- 护航舰发射的不是决定性弹药，而是一组迫使敌方火控分心的时间钉。
- 母舰回收窗口被迫延后，整支突入队因此从进攻资源变成了待救援资源。
- 主力舰仍然完好，但它的火控、护盾、通信和冷却窗口已经被切成互不兼容的碎片。
