# 联邦人族命名规则候选

状态：候选，暂存规则
定位：用于后续重命名 `泰拉星`、`泰拉人族` 及其个体姓名规则的工作台草稿，不替代当前正式口径。
基准：`10_世界观正文/00_总纲/以太纪元世界观总纲.md`、`10_世界观正文/10_专题设定/联邦思潮状态与制度底线.md`、`10_世界观正文/10_专题设定/术语表.md`

## 一、使用边界

- 本文件只记录候选命名规则，暂不把 `泰拉星`、`泰拉人族`、`基准泰拉亚种` 等正式条目改名。
- 后续若选定新母星名、族群名和亚种名，需要同步总纲、文明圈对照表、联邦思潮专题、术语表、展示发布层和版本记录。
- 新命名规则不改变现有设定口径：联邦系主体族群仍是复合人族谱系，各亚种差异不能写成固定职业命运、天然等级或文明短板拼装。
- 个体命名可体现地方传统、谱系来源、殖民地历史和家庭选择，但不得作为公民权、职业资格或能力信用的判断依据。

## 二、总规则

后续重命名联邦人族时，可以采用“三轨并存”的命名体系：

| 轨道 | 用途 | 适用范围 |
| --- | --- | --- |
| 中式命名轨 | 稳重、公共登记感强，适合核心星区、公共机构、法庭、舰队正式档案。 | 母星核心区、公共法传统强的成员星区、军政与公共服务系统。 |
| 日式命名轨 | 地方谱系感、学园都市感和动画角色辨识度更强。 | 青丘地方谱系、旧殖民地、观测站、学院城、部分地方自治共同体。 |
| 日式西方生造名轨 | 按日语音拍和罗马音习惯生成的外来风动画名，不等同现实西方姓名。 | 边境星港、跨族群家庭、舰队学院、混合殖民地、面向角色塑造的主角型姓名。 |

三条命名轨可以并存，但不应机械拼贴。一个角色通常只选择其中一条作为主要登记名；必要时可拥有地方名、舰队通称或外文转写名。

## 三、中式命名轨

### 规则

- 格式以 `姓 + 双字名` 为主，少量可用 `姓 + 单字名`。
- 姓氏可以来自古老家系、殖民地登记、公共法庭、船坞共同体、成员星区或地方谱系。
- 名字可使用清晰、好读、有角色辨识度的汉字，但避免过度古风、宗教化或神秘化。
- 适合写入正式公民登记、军校档案、法庭听证、公共机构任命和舰队人员名单。

### 示例

| 类型 | 示例 |
| --- | --- |
| 核心区登记名 | 林月桥、纪星澜、沈知衡、白承序、陆遥川 |
| 公共机构或法庭感 | 许闻序、程照临、韩明砚、顾承安、苏知遥 |
| 舰队或边境登记名 | 岑望舟、江远澜、周启航、白临川、陆星野 |

### 注意

- 该轨道保留较强中文语感，适合当前文档语境和联邦公共法传统。
- 若后续希望降低现实文化感，应减少现实常见姓氏密度，或把中式轨限定为某些地方谱系。

## 四、日式命名轨

### 规则

- 格式以 `苗字 + 名` 为主。
- 苗字可带自然、星港、观测、学院或地方共同体意象。
- 名可以更短，偏一到两音节，便于角色称呼。
- 适合青丘地方谱系、学园都市、旧殖民地、观测站和部分动画感角色。

### 示例

| 类型 | 示例 |
| --- | --- |
| 地方谱系名 | 星见澪、七濑遥、朝仓灯、白羽凛、远野知衡 |
| 学院或观测站风格 | 月城绫、雾岛司、风见悠、浅羽澪、星野律 |
| 青丘地方谱系风格 | 白羽狐月、星见铃、七濑璃、朝仓穗、远野澪 |

### 注意

- 日式命名轨可以作为地方谱系传统，不应覆盖整个联邦人族谱系。
- 避免让命名轨暗示青丘亚种或其他亚种具有固定职业、固定性格或天然能力命运。

## 五、日式西方生造名轨

### 定义

这里的“日式西方生造名”不是把日式姓氏和现实西式名字拼接在一起，而是指动画中常见的片假名外来风生造名：读起来像异国名或西方风名字，但实际按日语音节、罗马音和动画命名习惯生成，现实西方语言通常不会这样拼写。

### 规则

- 底层音拍以 `CV`、`CVV`、`CVCV`、`CVCVCV` 为主，少用真实英语、法语、德语中的复杂辅音簇。
- 常用音节包括 `a`、`i`、`e`、`o`、`ri`、`ru`、`ra`、`ia`、`aria`、`eria`、`fina`、`sena`、`noa`、`ruka`、`lilia` 等。
- 可保留 `l / r` 的转写摇摆，例如 `Liria / Riria`、`Luka / Ruka`，中文正文中统一音译即可。
- 避免真实西方姓氏后缀和贵族前缀，例如 `-son`、`-berg`、`-stein`、`Mc-`、`von`、`de`。
- 正式文本中优先写中文音译；角色设定卡或附表可补罗马音。
- 这种名字应像“被当地文字系统音译过的异国风姓名”，不是现实英语、法语或德语姓名。

### 示例

| 中文音译 | 罗马音候选 | 备注 |
| --- | --- | --- |
| 莉莉娅 | Liria / Riria | 柔和主角型。 |
| 瑟莉娅 | Seria | 学院、医疗、导航角色可用。 |
| 诺艾尔 | Noeru | 外来风明显，但按日语音拍生成。 |
| 露卡 | Ruka / Luka | 短名，适合舰队或边境角色。 |
| 艾莉娜 | Erina | 轻动画感，现实感较低。 |
| 菲莉娅 | Firia | 可用于跨族群家庭或边境星港。 |
| 塞娜 | Sena | 短促、好读。 |
| 米莉卡 | Mirika | 更明显的动画风生造名。 |
| 莉法 | Rifa / Lifa | 简短角色名。 |
| 阿莉娅 | Aria | 可用，但需避免过度泛用。 |
| 卡莲娜 | Karena | 比现实西方名更音译化。 |
| 露维娅 | Ruvia / Luvia | 柔和、异国风。 |

### 注意

- 不要把这一轨写成“西方文化姓名”。它是动画风外来名，不对应现实西方语言或民族。
- 不建议与现实日式苗字硬拼成一个姓名，例如“莉莉娅·星见”只适合作为特殊混合登记，不应作为该轨默认格式。
- 若角色需要完整登记，可使用 `个人名 + 来源支名`、`个人名 + 公民节点` 或只保留单名，再在档案字段中记录支系和籍贯。

## 六、登记与角色使用建议

| 场景 | 推荐命名轨 | 说明 |
| --- | --- | --- |
| 总纲、术语表、文明圈专题 | 中性正式族群名另定 | 不直接使用个体姓名规则替代族群名。 |
| 公共机构、法庭、舰队正式档案 | 中式命名轨 | 保持清晰、正式、便于中文文档阅读。 |
| 青丘地方谱系或学院城角色 | 日式命名轨 | 可强化地方传统和动画辨识度。 |
| 边境星港、跨族群家庭、主角型角色 | 日式西方生造名轨 | 可提高角色轻快度和异域感。 |
| 混血或跨星区角色 | 任选一条主轨，另设通称 | 不建议把多个轨道机械拼成一个名字。 |
