# 灵脉谱系设定

状态：采用  
基准：`10_世界观正文/00_总纲/以太纪元世界观总纲.md`、`能力系统索引.md`、`个体能力运行与成长机制.md`、`灵脉结构原理与工程演变案例.md`、`灵脉与灵械标准化体系.md`

本文件是“以太纪元”灵脉谱系的正式专题入口。个体能力运行与成长机制负责说明灵脉如何在个体身上运行；灵脉结构原理与工程演变案例负责说明内部结构如何具体改进；标准化体系负责说明灵脉如何被工程化、认证和定制；本文件负责说明某个功能工程技术方案如何从粗糙原型发展为成熟主干，并在任务压力、文明制度和应用场景中分叉、融合、断代或衍生新功能。

## 一、谱系定位

灵脉谱系是对灵脉工程技术方案演变脉络的记录，不是新的固定能力分类，也不是从弱到强的功能等级表。它用于追踪一条灵脉从粗糙原型发展为成熟主干的过程：最初解决什么问题，早期方案有哪些缺陷，后续用哪些结构改造解决缺陷，又因哪些应用场景衍生出稳定分支。

谱系只说明工程继承关系，不直接决定灵脉强弱、稀有度、标准化状态或定制化状态。同一条谱系可能同时存在标准教学分支、军用改型、家族封存分支和黑市旁支；同一功能也可能由不同谱系独立实现。功能分级和应用场合仍然存在，但它们应被写成推动方案迭代的压力，而不是谱系本体。

灵脉谱系口径必须区分四组概念：

- 功能分类：说明灵脉做什么，例如感知、认知、操控、生理或特殊功能。
- 应用场景：说明灵脉在哪些任务、职业、环境或制度约束下使用。
- 标准状态：说明灵脉是否公开、可认证、可维护、可教学、可交易。
- 谱系关系：说明灵脉从哪种工程方案来，继承了什么关键结构，修正了哪些旧缺陷，又分化出哪些功能或风险。

因此，红外视觉可以被归入感知类，也可以用于民用安全、侦察、医疗筛查或军用识别；这些功能和场景本身不是谱系。只有当文本说明它从低分辨率热源感知方案发展到滤波、过曝保护、视觉转译、多谱段融合和接口输出时，才是在写赤谱感知这类工程谱系。

## 二、核心术语

| 术语 | 定义 | 使用场景 |
| --- | --- | --- |
| 原型灵脉 | 一条谱系中最早被稳定记录、复现或追溯的工程方案源头，通常粗糙、低精度或风险较高。 | 说明后续分支继承的基础逻辑和早期缺陷。 |
| 谱系主干 | 由原型灵脉延续出的主要工程改造路线，通常保留核心结构、训练方式和维护方法。 | 说明某类灵脉如何从简易方案发展为主流教学、维护或研究路径。 |
| 分支灵脉 | 从主干中分化出的稳定改型，已形成明确适配对象和维护方式。 | 用于职业、文明、家族或任务差异。 |
| 旁支改型 | 偏离主干较多，但仍能追溯到同一原型逻辑的改造版本。 | 用于黑市改装、个人定制或地方工坊版本。 |
| 衍生功能 | 某条谱系在解决旧缺陷或适配新场景时形成的附带能力方向。 | 用于解释为什么一条感知、接口或防护谱系会长出诊断、审计、隔离或调度能力。 |
| 封存旧路线 | 因维护成本、事故风险、制度限制或被新方案替代而停止普及的分支。 | 用于保留历史技术、家族秘传、战时复启或剧情冲突。 |
| 断代谱系 | 原型资料、维护方法或训练路径中断，只能通过残留结构复原的谱系。 | 用于遗产、失传家族技术和灾后残存灵脉。 |
| 遗产接枝 | 将古文明蓝图、深层以太回响或未知模块接入现有灵脉谱系的行为。 | 用于高风险能力跃迁和文明伦理冲突。 |
| 谱系污染 | 分支引入异常结构后，反向影响同源灵脉或接口系统的风险。 | 用于解释兼容失败、回响感染和肃正协议警戒。 |

“原创灵脉”只描述设计来源，不等于新谱系已经成立。只有当它的原型逻辑被记录、复现，并出现可追踪的分支、维护方法或教学资料时，才可以被整理为一条谱系。

## 三、发展机制

灵脉谱系通常通过以下机制形成：

- 原型捕获：某些种族、家族、训练团体或特殊环境中出现可重复的能力结构，最初只解决狭窄问题，例如看见大块热源、接入单件灵械或维持低功率隔离膜。
- 缺陷暴露：原型进入训练、职业或事故场景后，会暴露低分辨率、误判、过曝、反馈过载、断连失败、污染倒灌或维护不可复现等工程缺陷。
- 结构改造：灵脉工程师、军校、工会、家族或文明机构会加入滤波、限流、反馈压缩、接口总线、调频、单向转译、沙箱隔离或战损降级等结构。
- 主干稳定：经过长期复测后，某些改造会沉淀为教材、认证、维护周期、失效处理和可移植铭刻方案，成为谱系主干。
- 应用分化：民用安全、舰队协同、医疗隔离、商盟授权、深层样本封存、黑市改装等场景会推动主干向不同约束分化。
- 衍生功能回流：分支在解决特定问题时，可能发展出原型没有的附带功能，例如维护诊断、残热追踪、合同审计、样本封存或工程排障；这些功能如果仍继承主干结构，就属于谱系衍生，而不是新功能分类直接变成新谱系。
- 封存与复启：维护成本过高、事故率过高或监管压力过大的旧路线会被封存；战时需求、遗产研究或地方工坊可能让封存路线重新出现。
- 断代与接枝：古文明蓝图、深层以太回响或灾后残留资料可能成为断代复原起点，也可能通过遗产接枝改变现有主干。

谱系发展不是单向进步。一条更精细的分支可能因维护链过重而不适合普及；一个粗糙旧方案可能因低依赖和低延迟在极端环境中被保留；遗产接枝可能带来能力跃迁，也可能造成污染、人格回响或肃正协议警戒。

## 四、谱系记录字段

记录一条灵脉谱系时，至少说明以下内容：

| 字段 | 说明 |
| --- | --- |
| 谱系名称 | 使用世界内名称，不用临时比喻。 |
| 功能起点 | 原型最初试图解决的狭窄问题，不直接等同于成熟功能分类。 |
| 原型逻辑 | 原型灵脉的核心输入、处理和输出是什么。 |
| 早期方案缺陷 | 原型为什么粗糙，具体表现为低精度、高误判、过载、难维护、污染风险或接口不稳。 |
| 关键结构 | 保持谱系身份的核心灵纹、灵环、灵节或灵段。 |
| 关键工程改造 | 从原型走向主干时新增的滤波、限流、调频、反馈压缩、沙箱、单向转译、维护诊断等结构。 |
| 发展来源 | 自然形成、训练诱导、工程铭刻、职业标准化、家族适配、任务分化或遗产复原。 |
| 迭代压力 | 哪些事故、职业任务、文明制度、市场需求或环境风险推动改造。 |
| 成熟主干 | 经过验证后被保留为教学、维护、认证或职业训练路径的方案。 |
| 衍生功能 | 由于工程改造和场景适配额外长出的稳定能力方向。 |
| 封存旧路线 | 被淘汰、限制、秘传或仅在特殊场景复启的旧方案。 |
| 主要分支 | 稳定存在的分支灵脉、旁支改型或断代残支，并说明它们继承或偏离了什么。 |
| 适配对象 | 适合哪些灵核参数、种族倾向、职业任务、灵械接口或文明制度。 |
| 公开程度 | 公开教材、军方封存、家族秘传、商盟授权、环带限制或遗产隔离。 |
| 维护体系 | 谁能维护，维护周期如何，故障如何定位、降级或断连。 |
| 兼容风险 | 同源兼容、频率冲突、接口污染、认知过载或谱系污染。 |
| 失效模式 | 失效时表现为降级、延迟、误读、烧蚀、污染、人格回响或外部劫持。 |

如果谱系尚未稳定，应标明“未成谱系”“断代待复原”或“实验分支”，避免把短期定制方案写成成熟传承。正式谱系示例至少应能写出一条因果链：原型缺陷如何暴露，工程改造如何解决问题，成熟主干如何保留核心结构，分支又如何因场景压力衍生新功能。

## 五、与标准化和定制化的关系

谱系关系可以帮助判断定制方案的结构来源和历史风险，但不能替代标准化审核。标准化审核看当前方案是否可教学、可维护、可追责；谱系记录则说明这个方案为什么长成现在的样子，以及它继承了哪些旧缺陷或封存风险。

- 标准灵脉可以来自成熟谱系，也可以来自曾经的原创设计。
- 定制灵脉可以基于成熟谱系改造，也可以来自尚未成谱系的原创方案。
- 参考灵脉通常属于某条谱系的原型、主干或分支。
- 遗产型参考灵脉可能属于断代谱系，也可能只是无法归谱的未知结构。

同源谱系通常更容易同步和维护，但不代表绝对安全。两个同源分支如果长期向不同任务压力演化，可能在运行频率、反馈强度、接口协议或污染隔离方式上产生冲突。相反，不同谱系的灵脉如果通过接口标准和调频结构处理，也可以稳定协同。

## 六、示例谱系

### 断代谱系与遗产接枝案例：先驱者蓝图

案例逻辑：先驱者蓝图或深层以太回响不是普通成熟谱系。现代文明拿到的往往是局部灵段、异常反馈、材料残片或污染后的运行痕迹，只能复原片段工程逻辑，不能把它当成可直接铭刻的完整升级路线。

现代复原链条：

- 残片识别阶段：研究者先标记仍能运行的灵纹、灵环或灵节，确认输入、处理和输出是否还能对应。此阶段的缺陷是原型目标不明，很多结构只能看出反馈强度，却无法判断原本功能。
- 隔离沙箱阶段：可疑灵段被迁入外置隔离灵械，只允许单向转译、低功率触发和人工复核。工程目标不是立即使用能力，而是避免未知结构进入灵核主回路。
- 低功率转译阶段：研究者将异常反馈转写为现代可识别的调频、限流、滤波或污染指标，并与已知谱系比对。能稳定解释的片段才可能归为断代残支，不能解释的继续标为未知结构。
- 联邦封存分支：把可复原片段作为军用与遗产研究资产保存，重点是风险审计、舰队级隔离、事故复盘和接触权限。
- 裂谷人体实验分支：尝试绕过部分沙箱，让改造个体直接适配异常反馈，可能获得高上限，也最容易引发人格回响、污染扩散和谱系污染。
- 商盟授权拆分分支：试图把安全片段拆成可授权模块，例如异常预兆识别、样本锁定或失效模型校准，但常被契约争议、灰色改装和肃正协议警戒限制。

衍生用途只能停在可验证片段层面。某个先驱者残段可以帮助改进污染筛查、异常预警或接口熔断，但只要完整输入、处理和输出链条没有复原，就不能写成成熟谱系主干。

结构细节：现代复原不会让残段直连灵核，而是先放入沙箱段，再用限流段、断连段和单向排出结构控制反馈。能被解释的信号才进入现代转译段，不能解释的信号继续封存。

维护边界：先驱者蓝图没有现代成熟主干，只有断代残支、隔离复原路径和接枝尝试。任何维护都必须以封存记录、沙箱环境、单向转译和人工复核为前提。

主要风险：断代程度越高，现代分支越可能误读原型逻辑。能力跃迁必须伴随识别困难、接口不兼容、污染风险或文明监管冲突，不能作为普通升级路线处理。

### 赤谱感知谱系

功能起点：早期热源感知训练。最初方案只要求使用者在黑暗、烟尘或低可见环境中察觉大块热差，不追求目标识别、医疗筛查或战术接入。

工程演变链条：

- 粗糙热源原型：通过赤谱感应环捕捉近距离热差，再把热源压成意识层亮斑。缺陷是分辨率低、背景热流容易误报，强热源会造成过曝和短时失明。
- 滤波保护改造：加入背景滤波环、过曝保护和低功率调控，能区分稳定热源、运动热源和杂散热流。这个阶段解决了训练安全问题，成为可教学主干的基础。
- 视觉转译主干：增加热差转译段，把亮斑转为轮廓、距离感和警戒色，并允许输出到神经视觉或灵械显示接口。它让赤谱感知从“粗略感觉热”变成可复检、可训练、可维护的工程方案。
- 民用外置分支：把主要感应段和放大段迁移到外置目镜，体内只保留提示、校准和疲劳保护。它牺牲反应速度，换取低侵入、易维护和可替换镜组。
- 侦察多谱段分支：为解决复杂地形误判，接入声波定位、残热追踪和环境滤波，衍生出残热路径复盘、生命热征筛查和伪装源识别等功能，但显著增加注意力窗口占用。
- 军用识别前端分支：将赤谱轮廓压缩成可被目标识别、弹道预测和战术网络读取的接口输出，依赖高权限接口、延迟校准和误伤责任记录。
- 黑市过敏旁支：非法提高感应阈值，能捕捉微弱热残留，但绕过过曝保护和疲劳提示，容易造成感知残留、误锁、神经疲劳和放大段烧蚀。

结构细节：本谱系的关键改进是把直连感应环改成“感应段 -> 分层滤波 -> 平衡放大环 -> 视觉转译 -> 疲劳提示”。这个改法降低了误报和过曝，但增加了延迟、维护点和注意力消耗。

封存旧路线：早期无滤波直感路线因过曝和误报事故被多数教学机构淘汰，只在少数家族训练或黑市改装中残留。

谱系身份：各分支都保留赤谱感应环和热差转译段，因此能追溯到同一原型；如果只使用战术网络进行目标识别，而不继承赤谱感应结构，就不属于本谱系。

维护与失效：主干维护重点是滤波和过曝保护；外置分支重点是镜组校准；军用分支重点是接口延迟和记录追责；黑市旁支常见问题是放大段烧蚀和感知残留。

### 通用接口谱系

功能起点：早期外置灵械接入。最初方案只要求灵核能发现一件外置模块，并接收基础反馈，不考虑多模块调度、授权审计或遗产隔离。

工程演变链条：

- 原型接入段：只能识别模块存在和传入反馈，缺陷是同步漂移难以发现，输出过强时会反灌灵核，异常模块也不一定能及时断连。
- 同步限流改造：加入身份识别、同步频率、反馈限流和手动断连，让接口从一次性接入变成可训练、可复测的基础职业灵脉。
- 接口总线主干：为解决多模块互相抢占通道的问题，加入优先级分配、反馈压缩、调频提示和紧急断连序列。成熟主干不再只服务单件灵械，而是让使用者安全调用一组外置模块。
- 舰队同步分支：强化多节点同步、指令校验、权限分级和战损断连，适合联邦舰队体系，并衍生出小队态势同步和战损权限转移。
- 灵械工坊维护分支：从接口总线中发展出诊断、调频、替换件识别和故障复测功能，适合灵械模块设计师和维修认证。
- 商盟授权分支：加入授权密钥、租赁时限、交易记录和售后追责结构，适合跨文明商品模块，并衍生出临时租用、远程停用和责任链审计。
- 遗产隔离分支：加入沙箱接入、单向转译、污染熔断和人工复核，用于未知蓝图和深层以太装置。它继承接口识别和断连结构，但把安全边界推到外置隔离层。

结构细节：本谱系的关键改进是把单件直连改成“识别段 -> 同步段 -> 限流段 -> 断连段 -> 统一接口”。统一接口让多件灵械可以排队、分级和压缩反馈，但也带来接口延迟和权限争议。

封存旧路线：早期直连式接口因反馈反灌和异常模块难断连而被标准体系淘汰，只能在低功率教学模型或隔离实验中保留。

谱系身份：各分支都继承识别、同步、限流、断连这组核心结构。只要某个灵脉只提供通讯或权限记录，而不承担灵核与外置模块的同步保护，就不能归入本谱系。

维护与失效：通用主干常见问题是同步漂移；舰队分支风险是指令串扰；商盟分支风险是密钥伪造；遗产隔离分支风险是接口倒灌和污染熔断失败。

### 玄庭推演谱系

功能起点：联邦早期航路规避训练。最初方案只要求指挥者在小规模舰队行动中压缩星体、残骸、以太扰动和友舰位置，找出相对安全路径。

工程演变链条：

- 航路规避原型：以空间定位、航迹残留和危险区域标记为核心，输出一条可行路径。缺陷是只适合短时局部态势，容易残留旧路径权重，并占用大量注意力窗口。
- 路径压缩改造：加入路径压缩、失败路径清理和基础注意力分屏，让使用者能同时处理航道、友舰和撤离方向。
- 威胁权重主干：为适应军校训练和舰队指挥，加入威胁权重、火力窗口、撤离顺序和责任记录，形成标准推演灵脉。成熟主干的核心不是“会推演”，而是把复杂空间态势压缩成可行动的权重结构。
- 远征舰队分支：强化长时间态势维持、跨舰同步、异常环境漂移修正和疲劳降级，衍生出远征态势维持、航图误差标记和锚点复检提示。
- 特战小队分支：裁剪大型舰队结构，保留短时路径预测、队友位置和近距威胁提示，适合登陆、突入和撤离。
- 工程排障衍生支：部分维护部门借用路径压缩和权重清理结构，用于复杂设备失效链分析。它能追溯到玄庭推演谱系，但不等于所有工程诊断都属于本谱系。
- 名门封存分支：部分军事家族保留旧式权重模型，适合特定训练风格和家族战例库，但与新式舰队接口兼容较差。
- 先驱者接枝实验：尝试把遗产蓝图中的异常预测段接入主干，可能提升未知态势识别，也可能引发人格回响和谱系污染。

结构细节：本谱系的关键改进是把单一路径建议改成“路径清理 -> 权重提示 -> 注意力分屏 -> 反馈压缩 -> 复检记录”。它不再替使用者做唯一判断，而是把复杂态势压成可行动提示。

封存旧路线：未清理失败路径残留的旧式推演训练会强化错误权重，现代军校不再作为主干教学，只允许作为历史战例或家族封存技术研究。

谱系身份：本谱系的核心不是“会推演”，而是继承航路规避原型中的空间权重、路径压缩和注意力分屏结构。医疗诊断、工程排障或商盟调度即使也需要推演，若没有继承这些结构，就不属于玄庭推演谱系。

维护与失效：主干需要清理旧战例权重和失败路径残留；远征分支需要监控长期疲劳和漂移修正；名门分支容易产生训练风格固化；遗产接枝实验需要沙箱和人工复核，不能直接作为标准升级。

### 霜膜护层谱系

功能起点：环带生态维护者保护微型生态舱的低功率隔离膜。最初方案只要求隔开污染、温差和小型冲击，维持脆弱生物样本的稳定环境。

工程演变链条：

- 生态舱隔离原型：低功率、稳定、可长期运行，能隔开温差和轻微污染。缺陷是展开慢、承压低，遇到快速冲击或未知污染时只能被动破裂。
- 污染识别改造：加入污染识别、生态阈值限制和缓慢修复结构，让护层能判断哪些扰动应隔离，哪些环境交换必须保留。
- 环带维护主干：把隔离、修复、阈值和维护复测整理为保守标准，强调长期稳定、低刺激和可追责维护，而不是瞬间承压。
- 医疗隔离分支：在主干基础上降低反馈刺激，用于病患、灵核裂纹和神经反噬后的低刺激防护，衍生出恢复舱护层和术后污染筛查。
- 深层封存分支：强化污染隔离、单向排放和样本链记录，用于遗产残片、深层以太样本和危险灵械。它把生态隔离方案扩展为样本封存技术。
- 联邦战场改型：为解决战场展开慢的问题，提高展开速度和抗冲击能力，增加战损降级和短时缓冲提示，但牺牲长期稳定和生态友好阈值。
- 黑市硬化旁支：强行提高承压上限，短时接近战术护盾，但绕过阈值限制和降级提示，容易导致护层破裂后污染反冲。

结构细节：本谱系的关键改进是把单层硬抗改成“稳相缓冲段 -> 污染识别 -> 多相导流环 -> 单向排出 -> 战损降级”。它让护层能区分冲击和污染，也让破裂前保留最低保护。

封存旧路线：只增厚护层、不识别污染类型的旧式硬膜路线被环带限制使用；它在短时承压上有价值，但不适合作为生态、医疗或深层封存主干。

谱系身份：各分支都继承霜膜式缓冲和污染隔离结构。纯粹依靠高吞吐输出硬抗冲击的战斗护盾不属于本谱系，最多只能借鉴其中的隔离段。

维护与失效：环带主干重视长期稳定和阈值复测；医疗分支重视低刺激反馈；深层封存分支重视单向排放和样本链记录；战场改型重视战损降级；黑市旁支常见问题是稳定环烧蚀和污染倒灌。
